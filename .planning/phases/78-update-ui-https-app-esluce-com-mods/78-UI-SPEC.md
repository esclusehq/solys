## Interaction Contract

### Modal Overlay Pattern (D-03, D-04, D-06, D-07)

Both modals (Version List modal and Add-to-Server modal) use a consistent cosmic-themed overlay pattern, derived from `InviteFriendsModal.jsx` lines 60-68 but updated to cosmic theme variables:

```jsx
<div className="fixed inset-0 bg-black/60 flex items-center justify-center z-50 p-4"
     onClick={onClose}  // backdrop click closes
     role="dialog" aria-modal="true">
  <div className="bg-[var(--color-nebula)] border border-[var(--color-cosmic-border)]
                  rounded-xl p-6 max-w-lg w-full space-y-4 shadow-xl"
       onClick={(e) => e.stopPropagation()}>
```

- **Title row:** Left-aligned heading (text-sm font-bold), right-aligned X close button (lucide `X` icon, text-muted hover:text-white)
- **Backdrop:** `bg-black/60`, click dismisses
- **Escape key:** Closes modal (add `useEffect` with `keydown` listener for Escape)
- **Max width:** `max-w-lg` for version list modal, `max-w-md` for server picker modal
- **Scroll:** Modal content scrolls internally if it exceeds viewport (`max-h-[80vh] overflow-y-auto`)

### Version List Modal (D-06, D-07)

- **Trigger:** Clicking **Versions** button on a mod result card
- **Content:** Scrollable list of version entries, each showing:
  - Version number (mono, text-sm, bold)
  - Release date (text-xs, text-muted)
  - Loader badges (compact chips: e.g. `[Forge]` `[Fabric]` — light gray bg, text-xs)
  - An **Install** button (cosmic-cyan solid, same style as primary CTA) on the right side of each row
- **No version selected by default** (per D-04)
- **Install button** in version row → triggers Add-to-Server modal flow (D-07)
- **Empty state:** If mod has no versions, show "No versions available for this mod." centered in modal
- **Loading state:** `EscluseSpinner` centered while versions are fetched from `modsApi.getVersions(mod.project_id)`

### Add-to-Server Modal (D-03, D-04, D-05)

This modal is used in two entry points:
1. **Clicking Add** on a mod result card → needs version selection + server selection
2. **Clicking Install** from a version row in the Version List modal → needs server selection only (version already chosen)

**Flow for entry 1 (card Add):**
1. Modal opens with two-step form:
   - Step 1: **Select Version** — dropdown or list of versions (same content as Version modal but as picker)
   - Step 2: **Select Server** — dropdown of user's servers (fetch from serverStore / server API)
2. **Install** button (primary CTA) enabled only when both version AND server are selected
3. On confirm → calls install API → shows success toast → closes modal

**Flow for entry 2 (version row Install):**
1. Modal opens with version pre-selected (displayed as disabled read-only field)
2. **Select Server** dropdown only
3. **Install** button enabled when server is selected
4. On confirm → calls install API → shows success toast → closes modal

**Server dropdown pattern:**
- Dropdown fetches servers from existing store/API
- Options show: server name + game type (truncated if long)
- If no servers available: show "No servers available. Create a server first." with disabled dropdown
- Styling matches existing filter selects in `ModBrowserPage.jsx` line 108-110: `bg-[var(--color-cosmic-card)]/60 border border-[var(--color-cosmic-border)] text-white rounded-lg`

**Install API call (D-05):** Calls `modsApi.install(serverId, modProjectId, versionId)` — needs a new API method added to `templatesApi.js` (POST to a new or existing endpoint). The UI contract assumes:
- Success: 200/201 response → toast "Installed {mod.title} to {server.name}"
- Failure: error response → toast "Failed to install {mod.title}. {error}"
- Loading: button shows spinner, disabled state during API call

### Category Filter (D-08)

- **Visual:** Dropdown select styled identically to existing version and loader filters (line 108-110 pattern)
- **Options:** Dynamically fetched from Modrinth categories API (or hardcoded fallback: All Categories, bukkit, mod, datapack, resourcepack, shader)
- **Default:** "All Categories"
- **Placement:** First in the filter bar row (before version and loader dropdowns)
- **Interaction:** Changing category triggers debounced search (same 300ms debounce as existing filters)

### Dynamic Game Versions (D-09)

- **Strategy:** Fetch available game versions from Modrinth API on **component mount** (`useEffect` on mount, not on filter interaction)
- **Fallback:** If API call fails, use hardcoded list: 1.21, 1.20.4, 1.20.1, 1.19.4, 1.18.2, 1.16.5 (existing hardcoded list)
- **Visual:** Same dropdown select styling as existing
- **Loading state during fetch:** Show static hardcoded list while fetching (non-blocking — existing list as placeholder)

### Page-Number Pagination (D-10)

Replaces the current **Load More** button. Visual layout:

```
[Previous]  1  2  3  4  5 ... 12  [Next]
```

- **Page size:** 25 (existing `pageSize` constant, unchanged)
- **Active page:** Cyan accent (`text-[var(--color-cosmic-cyan)] font-bold`, optional underline or background highlight)
- **Inactive pages:** `text-[var(--color-text-muted)] hover:text-white`
- **Previous/Next buttons:** `text-sm text-[var(--color-text-muted)] hover:text-white`, using lucide `ChevronLeft` / `ChevronRight` icons
- **Disabled state:** Previous disabled on page 1, Next disabled on last page — both with `opacity-40 cursor-not-allowed`
- **Ellipsis:** Show `...` when page range exceeds visible window (show max 7 page numbers: first, last, 5 around active)
- **Total pages calculation:** `Math.ceil(total / pageSize)`
- **Placement:** Centered below results grid, with `mt-6 gap-1.5 flex items-center justify-center`
- **URL state:** Page number stored in component state (not URL), resets to 1 on any filter/search change

### Filter Bar Layout

Row layout (left to right):

```
[ Search Input (flex-1) ] [ Category ▼ ] [ Version ▼ ] [ Loader ▼ ]
```

- All existing filter dropdowns (Version, Loader) preserved in place
- New Category filter prepended as the first dropdown
- All elements use same styling: `px-4 py-2.5 rounded-lg text-sm bg-[var(--color-cosmic-card)]/60 border border-[var(--color-cosmic-border)] text-white`
- Labels use `text-xs text-[var(--color-text-muted)]` above each dropdown (or placeholder-based labels inside the select)
