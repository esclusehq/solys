# Key interfaces already implemented:

From api/src/domain/auth/service.rs:
- JwtService: generate_access_token(), generate_refresh_token(), validate_token()
- Claims: sub, user_id, tenant_id, role, exp, iat
- TokenResponse: access_token, refresh_token, token_type, expires_in

From api/src/domain/auth/middleware.rs:
- AuthUser: user_id, tenant_id, email, role, is_admin()
- AuthError: InvalidToken, MissingToken
- FromRequestParts<S> for AuthUser extracts from Authorization header or cookies

From api/src/presentation/responses/api_response.rs:
- ApiResponse<T>: success, data, error, request_id
- ApiError: code, message, status_code, details, action
- ApiResponse::success(data) returns {success: true, data: data, error: null}

From api/src/presentation/handlers/auth_handlers.rs:
- POST /register: Validate email/password, hash password, create user, return JWT
- POST /login: Validate credentials, return JWT in httpOnly cookies
- POST /refresh: Generate new tokens from existing JWT
- POST /logout: Delete auth cookies
- GET /me: Return current user info
