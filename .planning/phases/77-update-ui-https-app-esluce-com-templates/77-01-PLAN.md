  <name>Task 3: Modify list_public_templates SQL to LEFT JOIN servers for usage_count</name>

  <action>
    Modify the `list_public_templates` method in `api/src/domain/server/template/repository.rs` to compute `usage_count` via a LEFT JOIN subquery on the `servers` table.

    **Open `api/src/domain/server/template/repository.rs`** and locate the `list_public_templates` method (line 246-275).

    Replace the entire SQL query (lines 248-256) with:
    ```sql
            SELECT t.id, t.game_type, t.category, t.display_name, t.description, t.config,
                   t.visibility, t.user_id, t.is_builtin, t.is_active,
                   t.created_at, t.updated_at,
                   COALESCE(usage_stats.usage_count, 0) AS usage_count
            FROM templates t
            LEFT JOIN (
              SELECT template_id, COUNT(*) AS usage_count
              FROM servers
              WHERE template_id IS NOT NULL
              GROUP BY template_id
            ) usage_stats ON usage_stats.template_id = t.id
            WHERE t.visibility = 'public' OR t.is_builtin = true
            ORDER BY t.is_builtin DESC, t.game_type, t.category
    ```

    **Why this change:** 
    - Table aliases (`t.`, `usage_stats.`) are needed because the subquery introduces a joined table
    - `COALESCE(usage_stats.usage_count, 0)` ensures templates with no servers show 0, not NULL
    - `WHERE template_id IS NOT NULL` filters out servers that weren't created from templates
    - `#[sqlx(default)]` on the Template struct ensures that other queries (list_templates, get_template, etc.) that don't SELECT usage_count get the default 0

    **No other SQL queries need modification.** The `#[sqlx(default)]` attribute on `usage_count: i64` in the Template model means sqlx will default to 0 for any query that doesn't explicitly SELECT usage_count. The other repository methods (list_templates, list_templates_by_game, get_template, get_template_by_id, create_template, update_template, list_templates_by_user) will continue working with their existing SQL.

    **Final compilation check:**
