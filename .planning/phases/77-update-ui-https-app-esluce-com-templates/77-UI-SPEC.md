## Interaction Contract

**Focal point:** The template card grid is the primary visual anchor. The "+ Create Template" button (top-right, cyan accent) is the primary action CTA. Secondary: filter tabs + search bar provide navigation and discovery context.

### Filter Tabs (D-01)
- **Visual:** Horizontal pill-shaped tab bar below the page header, above the filter bar. Three tabs: "Featured", "Yours", "All".
- **Styling:** Each tab is a `px-4 py-2 text-sm rounded-lg` pill. Active tab uses `bg-[var(--color-cosmic-cyan)]/10 text-[var(--color-cosmic-cyan)] border border-[var(--color-cosmic-cyan)]/30`. Inactive tabs use `text-[var(--color-text-muted)] hover:text-white hover:bg-[var(--color-cosmic-card)]`.
- **Accessibility:** Uses `role="tablist"` on the container, `role="tab"` on each button, `aria-selected={isActive}`.
- **Persistence:** Not persisted (session-only, D-04). Resets to "All" on page reload.
- **Data behavior:**
  - **Featured:** Shows `t.is_builtin === true` templates only
  - **Yours:** Shows `t.is_builtin === false` (user-created) templates only. Hidden when `user` is not logged in (greys out or shows empty with login prompt — see Copywriting: Yours tab empty state).
  - **All:** Shows both merged (current default behavior)
- **Placement:** Left-aligned in a row below the page title, above the search/filter bar. Horizontal gap: `gap-2`.

### Sort Controls (D-03)
- **Visual:** Dropdown select styled identically to the existing game type filter in `TemplateLibraryPage.jsx` — same `bg-[var(--color-cosmic-card)]/60 border border-[var(--color-cosmic-border)] text-sm px-4 py-2 rounded-lg` pattern.
- **Options:** "Name A-Z" (default), "Last Updated", "Most Used".
- **Persistence:** Not persisted (session-only, D-04). Resets to "Name A-Z" on page reload.
- **Placement:** Rightmost in the filter bar row, after the category filter.
- **Sort behavior:**
  - **Name A-Z:** Alphabetical by `display_name`
  - **Last Updated:** Descending by `updated_at` or `created_at`
  - **Most Used:** Descending by `usage_count` (servers created from this template)

### Category Filter (D-03)
- **Visual:** Dropdown select styled identically to the game type filter.
- **Options:** "All Categories" (default), plus derived category values from the template data set.
- **Persistence:** Not persisted (session-only, D-04). Resets to "All Categories" on page reload.
- **Placement:** Between the game type filter and sort dropdown in the filter bar row.
- **Data source:** Derived from `[...new Set(templates.map(t => t.category).filter(Boolean))]` — live from current data set.

### Game Type Filter (existing — restyle)
- **Options:** "All Games" (default), plus unique `game_type` values derived from the data.
- **Styling:** Restyle to match cosmic theme consistency if not already aligned. Currently uses the correct pattern — verify.
- **Placement:** Leftmost select in the filter bar row, immediately after search input.
- **No changes from current behavior** beyond restyling.

### Search Input (existing — keep)
- **Behavior:** Client-side filter on `display_name`, `game_type`, and `category` — matching existing `TemplateLibraryPage.jsx` lines 43-47.
- **Styling:** Restyle to match cosmic theme consistency if not already aligned. Currently uses the correct pattern — verify.
- **Placement:** Leftmost in the filter bar row, first element.

### Filter Bar Layout (D-03, D-04)
- **Visual:** Single horizontal row containing: `[Search input (flex-1)] [Game type select] [Category select] [Sort select]`
- **Responsive:** On screens < 768px, reflows to two rows: first row = search (full width), second row = select dropdowns (each flex-1).
- **Spacing:** `gap-3` between elements, matching existing `TemplateLibraryPage.jsx` line 71.
- **Margins:** `mb-6` below filter bar to cards grid, matching existing line 71.

### Template Card Enrichment (D-02)
Existing card structure (from `TemplateCard.jsx`) gets three additional info elements:

1. **Version tag:** Displayed as a small badge, top-right corner area (before Official/Coming Soon badges). Shows `template.version` if present, else hidden. Styling: `px-2 py-0.5 text-[10px] rounded-full bg-gray-600/20 text-gray-400 font-mono`.
2. **Last updated date:** Displayed below description, formatted as relative time ("2 weeks ago", "3 days ago", "Yesterday"). Shows `template.updated_at` or `template.created_at`. Styling: `text-[10px] text-[var(--color-text-muted)]` with lucide `Calendar` icon (12px, `mr-1`).
3. **Usage count:** Displayed next to last updated. Shows `template.usage_count ?? 0` with lucide `Users` icon (12px, `mr-1`). Styling: `text-[10px] text-[var(--color-text-muted)]`.

Card layout (updated order, top to bottom):
1. **Header row:** Display name (left), badges section (right: version tag | Official | Coming Soon)
2. **Meta row:** `game_type / category` — existing
3. **Description:** 2-line clamp — existing
4. **Metadata row (new):** Calendar icon + "Last updated {relative}" · Users icon + "{N} servers" — gray muted text
5. **Actions row:** Create Server link (left), Edit / Delete links (right, when `canEdit`) — existing

### Delete Confirmation (replaces `confirm()` dialog)
- **Visual:** Cosmic-themed modal overlay using existing glass-panel pattern. Centered `max-w-md` container with backdrop blur.
- **Content:** 
  - Heading: "Delete Template?" (text-white font-bold text-lg)
  - Body: "Are you sure you want to delete **{template.display_name}**? Servers already created from this template will not be affected." (text-sm text-[var(--color-text-muted)])
  - Actions: "Cancel" button (secondary style) + "Delete" button (destructive red style)
- **Interaction:** Delete button triggers `onDelete(template.id)`, which calls `templatesApi.delete(id)` then removes from local state. Cancel closes modal.
- **No separate component needed** — can use inline modal state in `TemplateLibraryPage.jsx` or a shared `ConfirmDialog` component if one exists. Check for existing confirm modal pattern before creating new.

- **Sections kept:** Basic Info + Configuration — no new sections
- **Back link:** `← Back` button above the form title — matching existing lines 98-99
- **No new fields** — D-07

### Empty State (per tab, filters active)
When the active filter tab + search + game type + category combination yields zero results:
- Show a centered block: heading + body text + optional CTA
- Featured tab empty: "No featured templates" / "Check back later"
- Yours tab empty: "No templates yet" / "Click + Create Template"
- All/any tab with active filters: "No matching templates" / "Try adjusting filters"
- See Copywriting Contract for exact copy

### Error State
When `templatesApi.list()` fails:
- Show inline error alert above the card grid (not a full-page error)
- Styling: `bg-red-500/10 border border-red-500/20 rounded-lg p-4 text-sm text-[var(--color-cosmic-red)]`
- Include a "Retry" button that calls `loadTemplates()`
- Existing data remains visible (don't clear templates array on error)
