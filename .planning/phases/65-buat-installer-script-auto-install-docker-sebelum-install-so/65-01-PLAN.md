# Phase 65 - Plan 1: Add container runtime auto-install to Solys installer

<objective>
Modify agent/solys/install.sh to detect and optionally install a container runtime (Docker or Podman) before downloading and installing the Solys agent binary. The installer checks for existing runtimes, installs missing ones, configures the Podman Docker-compatible socket, and generates a config.toml with the correct socket path.
</objective>

---

<tasks>

### Task 1: Add container runtime detection functions

| Field | Value |
|-------|-------|
| **Type** | modify |
| **Files** | `agent/solys/install.sh` |
| **Action** | Add `check_docker()` and `check_podman()` functions after existing color helpers. |
| **Verify** | Script runs `docker --version` + `docker info` (D-10) and `podman --version` + socket check (D-11). |
| **Acceptance Criteria** | Both functions return 0 if runtime found/working, 1 otherwise. |

**Implementation notes:**
- `check_docker()`: Runs `docker --version` then `docker info` to confirm daemon is responsive. If `docker` binary exists but daemon isn't running (docker info fails), try `systemctl start docker` then retry (D-12).
- `check_podman()`: Runs `podman --version` then checks for active Docker-compatible socket. Rootful: check `/run/podman/podman.sock`. Rootless: check `/run/user/$UID/podman/podman.sock`. If socket exists but inactive, `systemctl start podman.socket` (or `systemctl --user start podman.socket`).
- Use existing `info()`/`ok()`/`warn()`/`fail()` helpers for output.
- Both functions set a global `RUNTIME_SOCKET_PATH` variable on success.

### Task 2: Add container runtime installation function

| Field | Value |
|-------|-------|
| **Type** | modify |
| **Files** | `agent/solys/install.sh` |
| **Action** | Add `install_docker()` function that downloads and runs the official Docker convenience script (D-04), with package-manager fallback. |
| **Verify** | Simulate install failure path; confirm fallback to apt/yum/dnf on known distros. |
| **Acceptance Criteria** | Docker is installed and running after function returns. |

**Implementation notes:**
- Download `get.docker.com` script to temp dir, run via `sudo sh get-docker.sh` (D-04).
- If curl download of convenience script fails OR the script itself fails, fall back to package manager:
  - Debian/Ubuntu: `apt-get install -y docker.io`
  - CentOS/RHEL: `yum install -y docker-ce`
  - Fedora: `dnf install -y docker-ce`
- After installation, run `systemctl enable --now docker` (D-07).
- Verify with `check_docker()` afterwards. If still failing, call `fail()` (D-15).
- Root check (D-08): If not root, print error and exit early.

### Task 3: Add Podman socket auto-configuration (D-03)

| Field | Value |
|-------|-------|
| **Type** | modify |
| **Files** | `agent/solys/install.sh` |
| **Action** | When Podman is detected/installed, enable the Docker-compatible socket via systemd. |
| **Verify** | Confirm socket activation runs for both rootful and rootless scenarios. |
| **Acceptance Criteria** | Podman socket is enabled and started after configuration. |

**Implementation notes:**
- Rootful: `systemctl enable --now podman.socket` → socket at `/run/podman/podman.sock`
- Rootless: `systemctl --user enable --now podman.socket` + `loginctl enable-linger` → socket at `/run/user/$UID/podman/podman.sock`
- Verify socket is listening before proceeding.
- Set `RUNTIME_SOCKET_PATH` to the detected socket path.

### Task 4: Integrate runtime check into main() (D-02, D-14)

| Field | Value |
|-------|-------|
| **Type** | modify |
| **Files** | `agent/solys/install.sh` |
| **Action** | Call runtime detection/installation in `main()` before `download_artifacts()`. |
| **Verify** | Trace the full execution flow: detect → install (if needed) → verify → proceed to download. |
| **Acceptance Criteria** | Script aborts with clear message if no runtime can be installed (D-15). |

**Implementation notes:**
- Insert `ensure_container_runtime()` in main() after `detect_platform` and `build_urls`, before `download_artifacts`.
- `ensure_container_runtime()` flow:
  1. Check Podman with `check_podman()` — if found and socket active, set as preferred (D-02).
  2. If Podman found but socket inactive, auto-configure socket (Task 3), then verify.
  3. If Podman not found, check Docker with `check_docker()` — if found and running, use it.
  4. If Docker found but daemon not running, auto-start it (D-12), then verify.
  5. If neither Podman nor Docker found, install Docker via `install_docker()` (Task 2).
  6. After Docker install, do final verification. If still failing, `fail()` with troubleshooting info (D-15).
- Always run detection (D-14) — no skip flag.
- No interactive prompts during install (D-16).
