<objective>
Build the relay gateway: a new Rust + Axum service deployed on a single AWS EC2 c6i.large in ap-southeast-1a. The gateway accepts outbound WSS tunnels from agents, authenticates them via HMAC-signed callbacks to the backend, and forwards raw TCP player connections on :25565 to the right yamux stream by source-IP lookup. The gateway is the only Phase 68 component that owns player-facing network exposure.

This is the first of two parallel wave-3 plans (the other is Plan 05: dashboard UI). It is intentionally one plan because the gateway is a single deployment unit; the source files and deployment files split naturally across 2 tasks.

       - `tunnel.rs` (≈180 lines): `pub async fn run_tunnel_session(ws: WebSocketStream, state: Arc<AppState>)`. Upgrades the WS, parses the first message as `TunnelConnect { server_id, subdomain, public_port, agent_public_ip, region }`, calls `auth::authorize` with the bearer token, opens yamux over the WS, sends TunnelConnect JSON on the control stream, registers in `Registry`, then loops: `client.next_stream().await` → for each inbound yamux stream, spawns `player::forward_to_tcp(stream, server_id, state.clone())`. On any error or disconnect, calls `registry.unregister(server_id)` and reports `tunnel-event: disconnected` to the backend.

       - `player.rs` (≈100 lines): `pub async fn run_player_listener(state: Arc<AppState>)`. TcpListener::bind on `state.config.server.player_bind`. For each accepted TcpStream: check `state.rate_limiter.check(source_ip)`; if not rate-limited, `state.registry.lookup_by_ip(source_ip)`; if Some(server_id), `forward_to_tcp(stream, server_id, state)`; else drop. `pub async fn forward_to_tcp(tcp: TcpStream, server_id: Uuid, state: Arc<AppState>)`: opens a new yamux stream on the tunnel handle's client, `tokio::io::copy_bidirectional(&mut tcp, &mut yamux_stream).await`.

       - `backend.rs` (≈100 lines): `pub struct BackendClient { base_url: String, hmac_secret: String, http: reqwest::Client }`. Methods: `pub async fn report_tunnel_event(&self, server_id: Uuid, event_type: &str, reason: &str) -> Result<(), GatewayError>`. Internally signs the request with `auth::sign_request` and sends POST to `/internal/relay/tunnel-event`.

       - `heartbeat.rs` (≈60 lines): `pub async fn run_heartbeat_watcher(state: Arc<AppState>)`. 30s `tokio::time::interval` ticker. Each tick: iterate `state.registry.iter()`. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent → Gateway WSS | Agent opens outbound WSS to `relay.esluce.net:443` (Caddy TLS 1.3 terminated, then proxied to gateway :8080). Gateway validates Bearer token by HMAC-signed POST to backend. |
| Player → Gateway TCP | Player opens raw TCP to `<subdomain>.play.esluce.net:25565` (NLB passthrough to gateway). Gateway looks up server_id by source IP. |
| Gateway → Backend | Internal HMAC-signed POSTs to `/internal/relay/authorize` and `/internal/relay/tunnel-event`. |
| Gateway → Redis | Nonce dedup and (optional) rate limit coordination. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-68-17 | Spoofing | Bearer token on WSS | mitigate | Token is a 122-bit UUIDv4 verified by HMAC-signed backend callback. Token never sent in cleartext (TLS 1.3). |
| T-68-18 | Spoofing | Player source IP | mitigate | Source IP is set by NLB (no client control). NLB preserves client IP via PROXY protocol or by being a passthrough TCP listener; the gateway trusts the kernel-reported peer address. |
| T-68-19 | Tampering | Player → yamux stream | mitigate | All traffic is inside TLS 1.3 (WSS) and yamux (control + data). |
| T-68-20 | Repudiation | Tunnel lifecycle events | mitigate | All `tunnel-event` reports are HMAC-signed and logged on both sides. Backend is the source of truth for `relay_status`. |
| T-68-21 | Denial of Service | Player TCP flooding | mitigate | Per-IP rate limit (100 req/min) via in-process token bucket. 1 active tunnel per server_id (D-21). |
| T-68-22 | Denial of Service | Reconnect storm | mitigate | Agent side has exponential backoff (Plan 02). Gateway side rejects duplicate `TunnelConnect` for the same `server_id` by dropping the older. |
| T-68-23 | Information Disclosure | TunnelConnect.agent_public_ip | mitigate | Only used for the in-process `by_agent_ip` DashMap; never logged, never returned to a player. |
| T-68-24 | Elevation of Privilege | Tunnel limit bypass | mitigate | `Registry::register` returns `TunnelLimit` (HTTP 429) if 1 tunnel already exists; the older tunnel is force-closed. |
| T-68-25 | Tampering | HMAC secret in env | mitigate | Secret loaded from AWS Secrets Manager, not committed to git. `.env.example` documents the env var name only, no value. |
