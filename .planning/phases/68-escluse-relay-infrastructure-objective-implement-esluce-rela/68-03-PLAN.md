<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dashboard → Backend | JWT-authenticated user calls GET/PUT `/api/v1/servers/:id/relay/*`. Backend verifies user owns the server (via existing tenant check). |
| Gateway → Backend (internal) | HMAC-signed POSTs to `/internal/relay/*`. Backend verifies HMAC + timestamp window + nonce uniqueness via Redis. |
| Agent → Backend WS | TLS 1.3 encrypted. Backend verifies `node_id` from session matches server's `node_id` for any state-modifying TunnelConnect. |
| Redis nonce store | 5-min TTL window. The same nonce can never be replayed within or across the window. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-68-09 | Spoofing | `/internal/relay/*` callers | mitigate | HMAC-SHA256 signature required; secret loaded from `GATEWAY_HMAC_SECRET` env var. |
| T-68-10 | Tampering | Internal request body | mitigate | HMAC is computed over the body bytes; any tampering invalidates the signature. |
| T-68-11 | Repudiation | Internal request replay | mitigate | Each request carries a unique nonce; nonce stored in Redis with `SET ... EX 300 NX`. Replayed nonce returns 401. |
| T-68-12 | Repudiation | Internal request timestamp drift | mitigate | Timestamp must be within 300s of server clock; else 401. |
| T-68-13 | Information Disclosure | `/api/v1/relay/*` response | mitigate | Same tenant-isolation as other server endpoints (handled by existing JWT middleware + `verify_server_ownership` in RelayService). |
| T-68-14 | Denial of Service | Internal endpoint flooding | mitigate | Redis-backed rate limit: 100 req/min per source IP. Implemented in `internal_relay_handlers::post_authorize` via `redis::INCR` + `EXPIRE`. |
| T-68-15 | Elevation of Privilege | Mode override | mitigate | `set_mode_override` validates mode against `("relay", "direct", "auto")`; invalid value returns 400. The `connectivity_mode_override` column has a CHECK constraint. |
| T-68-16 | Spoofing | TunnelConnect from agent | mitigate | `on_tunnel_connect` calls `verify_server_ownership(node_id, server_id)` before mutating any state. |

## ASVS L1 Mappings (Phase 68 backend tier only — full coverage lives in plans 02, 04, 05)

- **V1.4 Access Control:** Tenant ownership check on every relay endpoint. Internal endpoints gated by HMAC, not JWT.
- **V2.1 Authentication:** HMAC-SHA256 with shared secret + nonce. JWT for user-facing endpoints (existing).
- **V3.7 Token Storage:** `relay_token` is stored in the database; never returned to the dashboard.
- **V4.1 Input Validation:** All enum fields validated against allow-list at both the entity layer and the database CHECK constraint.
- **V6.2 Cryptographic Practices:** HMAC-SHA256, constant-time comparison via `hmac::Mac::verify_slice`.
- **V7.1 Logs:** Every TunnelConnect/Disconnect/Heartbeat logged via `tracing::info!` with structured fields.
- **V9.1 Rate Limiting:** 100 req/min per source IP at the internal endpoint layer.
