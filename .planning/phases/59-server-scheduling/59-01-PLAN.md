## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-59-01 | Tampering | cron_task_handlers.rs :: create_task | mitigate | Validate task_type against whitelist array (["backup","restart","stop","command","start","sleep"]) — enforced in both create_task and update_task. Returns `Err("Invalid task_type")` if out of range. |
| T-59-02 | DoS | cron_task_handlers.rs :: create_task | mitigate | Cron expressions that fire every minute could cause excessive DB/Worker load. Since `next_run` is set to NOW() + cron_offset on creation, and Worker poll has a 30s interval, the rate limit is already bounded. Accept-minimum-interval validation is deferred (the agent's discretion). |
| T-59-03 | Information Disclosure | create_task/list_tasks response | accept | last_result and last_error are returned to the task owner (authorized via server ownership check and server.user_id == auth_user.tenant_id). No sensitive data exposure. |
| T-59-04 | Elevation of Privilege | cron_task_handlers.rs | mitigate | Every handler checks ownership via `server.user_id != auth_user.tenant_id` and `task.user_id != auth_user.tenant_id` — existing pattern preserved. |
