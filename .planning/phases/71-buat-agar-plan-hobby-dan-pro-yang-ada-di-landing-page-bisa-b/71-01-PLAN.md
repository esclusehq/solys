**Auth gate checkout handler (per D-05: unauthenticated → signin with plan context; D-06: authenticated → checkout API; D-07: default billing cycle is monthly):**
```typescript
const handleSubscribe = async (plan: Plan, cycle: 'monthly' | 'yearly') => {
  if (plan.name === 'Free') {
    navigate('/signin');
    return;
  }
  const { isAuthenticated } = useAuthStore.getState();
  if (!isAuthenticated) {
    // Save to sessionStorage for OAuth resilience (Pitfall 2)
    sessionStorage.setItem('intended_plan', JSON.stringify({ plan: plan.name, planCycle: cycle }));
    navigate('/signin', { state: { plan: plan.name, planCycle: cycle } });
    return;
  }
  try {
    const response = await billingApi.createCheckout(plan.name, cycle);
    const checkoutUrl = (response as any)?.checkout_url || response?.checkout_url;
    if (checkoutUrl) {
      window.location.href = checkoutUrl;
    }
  } catch (err) {
    console.error('Checkout error:', err);
  }
};
```

|-----------|----------|-----------|-------------|-----------------|
| T-71-01 | Tampering | URL params `?plan=`, `?plan_cycle=` | accept | Backend validates plan_id existence. Tampering causes at most a 404 error from checkout API. No data exposure. |
| T-71-02 | Spoofing | Fake "Current Plan" badge | accept | Backend `GET /api/v1/billing/subscription` requires `VerifiedUser` middleware. Only authenticated user's subscription data is returned. Frontend displays what backend returns. |
| T-71-03 | Information Disclosure | Subscription data API | mitigate | Backend uses `VerifiedUser` extractor (cookie-based auth with SameSite cookies). Frontend `apiClient` uses `withCredentials: true` so cookies are sent. CSRF-protected by existing SameSite configuration. |
| T-71-04 | Spoofing | `?checkout=success` URL param | accept | Parameter is cosmetic only — no backend call triggered. URL is cleaned with `window.history.replaceState()` immediately after detection. No authorization or data exposure risk. |
| T-71-05 | Information Disclosure | Lemon Squeezy API keys | mitigate | All Lemon Squeezy API calls happen server-side in the backend (Rust/Axum). Frontend never sees API keys or secrets. Frontend only receives the checkout URL. |
