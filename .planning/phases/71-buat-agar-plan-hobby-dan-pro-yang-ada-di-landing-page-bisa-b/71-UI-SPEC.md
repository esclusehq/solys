## Interaction Contracts

### I-01: Pricing Section — Plans API Fetch

| Property | Value |
|----------|-------|
| Trigger | PricingSection component mount |
| Action | `GET /api/v1/billing/plans` via `billingApi.fetchPlans()` |
| Loading state | Block section with centered `<Spinner>` (no skeleton cards) |
| Success | Render plan cards from API response data |
| Error | Show toast "Failed to load plans" + render hardcoded fallback data |
| Refresh | Once per mount. No cache, no SWR |
| **Source** | D-01, D-02, D-03, D-04 |

### I-02: Billing Cycle Toggle — Monthly/Yearly

| Property | Value |
|----------|-------|
| Default | Monthly (D-07) |
| Visual | Segmented toggle above plan cards: `[Monthly ●────○ Yearly]` |
| On toggle | Update displayed price to corresponding `price_{cycle}` |
| Savings badge | When yearly selected, show green savings badge: `Save ~17%` |
| Computation | `Math.round((priceMonthly * 12 - priceYearly) / (priceMonthly * 12) * 100)` |
| State drives | `billingCycle` variable: `"monthly"` or `"yearly"` (passed to checkout) |
| **Source** | D-09, D-10, D-11 |

### I-03: Checkout Trigger — Auth Gate

| Property | Value |
|----------|-------|
| Trigger | Click "Start for Hobby" or "Upgrade to Pro" button |
| Check | `useAuthStore.getState().isAuthenticated` |
| **If NOT authenticated** | `navigate('/signin', { state: { plan: planName, planCycle: cycle } })` AND save to `sessionStorage` for OAuth fallback |
| **If authenticated** | `POST /api/v1/billing/checkout { plan_id: planName, billing_cycle: cycle }` → receive `{ checkout_url }` → `window.location.href = checkout_url` |
| Error handling | Show toast on checkout API failure |
| **Source** | D-05, D-06 |

### I-04: SignIn Page — URL Param Handling

| Property | Value |
|----------|-------|
| Entry point | User lands on `/signin?plan=hobby&plan_cycle=monthly` (or with `location.state`) |
| Reading params | Check `location.state?.plan` first, then `URLSearchParams.get('plan')`, then `sessionStorage` |
| After login success | Call `billingApi.createCheckout(plan, planCycle)` → redirect to Lemon Squeezy |
| OAuth resilience | Plan context saved to `sessionStorage` before redirect to `/signin` — read in `OAuthCallback.tsx` after OAuth completes |
| **Source** | D-05 pattern, Pitfall 2 from RESEARCH.md |

### I-05: Post-Checkout — Dashboard Welcome Modal

| Property | Value |
|----------|-------|
| Trigger | Dashboard mount with `?checkout=success` URL param |
| Detection | `useSearchParams().get('checkout') === 'success'` |
| URL cleanup | `window.history.replaceState({}, '', '/dashboard')` immediately |
| Modal content | Plan name, features unlocked, limits, "Start creating servers" CTA |
| Dismiss | Click CTA button or click outside modal |
| If `?checkout=canceled` | Show toast: "Checkout canceled. You can try again anytime." + do NOT show modal |
| **Source** | D-08, D-12 |

### I-06: Current Plan Badge (Logged-In Users)

| Property | Value |
|----------|-------|
| Trigger | PricingSection mount when `isAuthenticated === true` |
| Action | `GET /api/v1/billing/subscription` via `billingApi.getSubscription()` |
| Matching | Compare `subscription.plan.name` to plan card name (lowercase) |
| Badge | Small "Current Plan" badge on matching card (top-right or below plan name) |
| Button swap | Replace "Start for Hobby"/"Upgrade to Pro" with "Manage" button |
| Manage action | `POST /api/v1/billing/portal` → `window.location.href = portal_url` |
| Error handling | Silently fail (no badge shown) — don't block pricing render |
| **Source** | D-13, D-14 |

### I-07: Free Plan Behavior

| Property | Value |
|----------|-------|
| CTA click | "Get Started Free" → `navigate('/signin')` |
| Behavior | Same as Phase 70 and before — sign-up creates user, no subscription record |
| No toggle effect | Free card shows `$0/mo` regardless of billing cycle toggle |
| **Source** | D-15 |
