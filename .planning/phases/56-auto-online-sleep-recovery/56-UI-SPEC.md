## Focal Point

The primary visual anchors in this phase are:

1. **Server action button (Sleep/Wake)** — in the ServerDetails header bar, replaces or augments the current Start/Stop button logic. This is the most prominent interactive element. When server is running with `auto_wake=true`, the stop action becomes "Sleep". When server is stopped with `auto_wake=true`, the start action becomes "Wake".

2. **Status badge (Sleeping)** — in ServerDetails info card and ServerManager list cards. Shows different visual than plain "Stopped" when `auto_wake=true`. Uses existing `StatusBadge.jsx` with new `sleeping` prop.

3. **Auto Sleep config panel** — in the ServerDetails Settings tab (below Discord Webhook section). Contains the toggle and timeout configuration. Secondary visual priority — configuration, not action.

Visual hierarchy flows top-to-bottom on the ServerDetails page:
- Header bar: Sleep/Wake action button, badge
- Tabs: Dashboard | File Manager | Plugins | Backups | Settings | Terminal
- Settings tab content: Discord Webhook (existing) → Sleep & Wake (new) → (existing content below)

#### 2a. Action Button — Sleep/Wake Toggle Logic

**Location:** Header bar `handleToggle` function (line 143) and button JSX (line 216-229).

**Modified `handleToggle`:**
```jsx
const handleToggle = async () => {
    try {
        if (server.status === 'running') {
            if (form.auto_wake) {
                await sleepServer(id);        // Sleep: stop + set auto_wake=true
            } else {
                await stopServer(id);          // Stop: plain stop
            }
        } else if (server.status === 'stopped' && server.auto_wake) {
            await wakeServer(id);             // Wake: start + reset auto_wake=false
        } else {
            await startServer(id);            // Start: plain start
        }
    } catch (err) { alert(err.message); }
};
```

**Button label matrix:**

| Current Status | `auto_wake` | Button Label | Button Style |

| running | true | 💤 Sleep | Red (stop action) — `border-[var(--color-cosmic-red)]` |

| stopped | true | 💤 Wake | Cyan (start action) — `border-[var(--color-cosmic-cyan)]` |
| stopped | false | ▶ Start | Cyan (start action) — `border-[var(--color-cosmic-cyan)]` |
| starting | any | ⏳ Starting... | Disabled |
| container_running | any | Starting Minecraft... | Disabled |
| stopping | any | Stopping... | Disabled |

```jsx
<section className="glass-panel p-6">

        </div>
    </div>

    {/* Sleep Timeout (visible only when toggle ON) */}
    {form.auto_wake && (
        <div className="mt-4">
            <label className="block text-xs font-bold text-[var(--color-text-muted)] mb-2">
                Sleep after (minutes of 0 players)
            </label>
            <input type="number"
                   value={form.sleep_timeout_minutes}
                   min={5} max={240}

            <p className="text-[10px] text-[var(--color-text-muted)] mt-2">
                ⏱ Server will auto-sleep after this many minutes with zero players.
            </p>

        disabled={saving}
        onClick={handleSaveSleepConfig}
        className="mt-5 w-full py-2.5 rounded-lg text-sm font-bold
                   bg-[var(--color-cosmic-cyan)]/10 text-[var(--color-cosmic-cyan)]
                   hover:bg-[var(--color-cosmic-cyan)]/20 border border-[var(--color-cosmic-cyan)]/30
                   disabled:opacity-50 transition-all">
        {saving ? 'Saving...' : 'Save Changes'}
    </button>
</section>
```

**Interaction:**
- Toggle OFF → hide timeout input, keep label visible
- Toggle ON → reveal timeout input with current value
- Timeout input: number, min=5, max=240, default=30
- Save → `updateServer(id, { auto_wake, sleep_timeout_minutes })` (exists via `useServers.js`)
- On save success: server state updates, UI reflects changes
- On save error: toast error message

**States:**

| State | Visual | Behavior |
|-------|--------|----------|
| **Toggle OFF** | Gray toggle, timeout input hidden | Auto Sleep disabled |
| **Toggle ON** | Cyan toggle, timeout input visible with value | Auto Sleep enabled, timeout configurable |
| **Saving** | Button shows "Saving...", disabled | Prevents double-submit |
| **Save Error** | Alert toast (same pattern as Discord webhook toast) | Section remains open with current values |
| **Save Success** | Alert toast (green) | Section reflects saved values |
