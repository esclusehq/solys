## Spacing Scale

Using Tailwind v4 default spacing scale (multiples of 4). Pre-populated from Phase 75 UI-SPEC + existing SettingsPage patterns:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, compact inline padding (`gap-1`, `p-1`) |
| sm | 8px | Badge padding, avatar sizes, compact spacing (`gap-2`, `p-2`, `w-8 h-8`) |
| md | 16px | Default element spacing, button padding (`gap-4`, `p-4`, `px-4 py-2`) |
| lg | 24px | Section padding, card internal padding (`p-6`, `gap-6`, `w-24 h-24` for avatar) |
| xl | 32px | Layout gaps between tab content sections (`gap-8`, `p-8`) |
| 2xl | 48px | Major section breaks, page header bottom margin (`mb-12`) |
| 3xl | 64px | Page-level outer padding (`p-16`) |

Exceptions: **none** — all existing SettingsPage UI conforms to Tailwind's 4px grid. The 1433-line file uses `p-6`, `mb-6`, `gap-6`, `gap-4`, `px-4 py-2` consistently.

## Typography

Pre-populated from Phase 75 UI-SPEC (same design system) + existing SettingsPage classes:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|

- Existing code used mixed weights (`font-medium` at line 541, `font-bold` at line 1403). Consolidate to `font-semibold` for all headings (section headings at lines 521, 541) and page title (line 1403). Only maintain the size distinction: `text-lg` for section headings, `text-2xl` for page title.
- Section heading color: `text-white` (kept as-is, already correct).

## Color

Pre-populated from `app/src/index.css` lines 4-19 (identical to Phase 75). The entire SettingsPage currently uses flat `bg-gray-700`, `bg-gray-800`, `border-gray-700`, `focus:ring-blue-500` — **all must be replaced** with cosmic theme equivalents:

| Role | Variable | Hex | Usage |
|------|----------|-----|-------|
| Dominant (60%) | `var(--color-deep-space)` | #080b15 | Page background (already correct — body inherits this) |
| Secondary (30%) | `var(--color-nebula)` / `var(--color-cosmic-card)` | #0d0f1a / rgba(255,255,255,0.03) | Tab content sections wrapped in `glass-panel`, card backgrounds, sidebar panels |
| Accent (10%) | `var(--color-cosmic-cyan)` | #0ddff2 | Active tab indicator, focus rings on all inputs, links, primary action buttons, status badge text |
| Destructive | `var(--color-cosmic-red)` | #ef4444 | Delete Account, Revoke API Key, Remove Member, Logout All Devices, Danger Zone accent, error states |
| Success | `var(--color-cosmic-green)` | #10b981 | Enable 2FA button, healthy webhook badge, session active dot |
| Warning | `var(--color-cosmic-orange)` | #f59e0b | Pending invite status, retry webhook button |
| Primary Blue | `var(--color-cosmic-blue)` | #3b82f6 | Save/Change/Test action buttons (kept as semantic alternative to cyan) |

**Accent reserved for:** Active tab bottom border (`border-b-2 border-[var(--color-cosmic-cyan)]`), focus rings on all inputs/selects/textareas, status badge backgrounds (`bg-[var(--color-cosmic-cyan)]/10`), status badge text, table row hover background, link hover states, drag-over avatar border.

**Replacement mapping (existing → cosmic):**

| Old Class | New Class |
|-----------|-----------|
| `bg-gray-700 rounded` | `bg-[var(--color-cosmic-card)] border border-[var(--color-cosmic-border)] rounded-xl` (for cards/panels) |
| `bg-gray-800 rounded` | `bg-[var(--color-nebula)] border border-[var(--color-cosmic-border)] rounded-xl` (for darker panels like delete confirmation) |
| `bg-gray-600` (disabled inputs) | `bg-[var(--color-nebula)] text-gray-500 cursor-not-allowed` |
| `border-gray-700` | `border-[var(--color-cosmic-border)]` |
| `focus:ring-2 focus:ring-blue-500` | `focus:ring-2 focus:ring-[var(--color-cosmic-cyan)]` |
| `focus:ring-red-500` (danger inputs) | `focus:ring-2 focus:ring-[var(--color-cosmic-red)]` |
| `text-blue-400` (tab active) | `text-[var(--color-cosmic-cyan)]` |
| `border-b-2 border-blue-400` (tab active) | `border-b-2 border-[var(--color-cosmic-cyan)]` |
| `text-gray-400 hover:text-white` (tab inactive) | `text-[var(--color-text-muted)] hover:text-[var(--color-text-main)]` |
| `hover:bg-[rgba(255,255,255,0.03)]` (table row) | Keep as-is (already using cosmic pattern) |

**⚠️ Note on `--color-bg-secondary`:** The CONTEXT.md references `bg-[var(--color-bg-secondary)]` for inputs, but this CSS variable does not exist in `index.css`. The correct replacement is `bg-[var(--color-cosmic-card)]` for input backgrounds. Do NOT reference `--color-bg-secondary` unless the variable is added first.

**Buttons — keep existing color semantics, only update Tailwind classes where needed:**
- Blue action buttons (`bg-blue-600 hover:bg-blue-700`) → Already use `bg-blue-600` which maps to `--color-cosmic-blue` — keep as-is, matches cosmic palette
- Red destructive buttons (`bg-red-600 hover:bg-red-700`) → Already use `bg-red-600` which maps to `--color-cosmic-red` — keep as-is
- Green success buttons (`bg-green-600 hover:bg-green-700`) → Already use `bg-green-600` which maps to `--color-cosmic-green` — keep as-is
- Gray secondary buttons (`bg-gray-600 hover:bg-gray-500`) → Use `bg-[var(--color-nebula)] border border-[var(--color-cosmic-border)] hover:bg-[rgba(255,255,255,0.06)]`
- Orange warning buttons (`bg-orange-600 hover:bg-orange-700`) → Already use `bg-orange-600` which maps to `--color-cosmic-orange` — keep as-is

**No light theme adjustments needed.** The existing `[data-theme="light"]` overrides in `index.css` lines 289-301 auto-apply.

## Interaction Contract

### Tab Bar (Existing — Cosmic Restyle Only)
- **Visual:** Horizontal tab bar directly below page heading "Settings". No sidebar conversion.
- **Active tab:** `text-[var(--color-cosmic-cyan)] border-b-2 border-[var(--color-cosmic-cyan)]` replaces `text-blue-400 border-b-2 border-blue-400`
- **Inactive tab:** `text-[var(--color-text-muted)] hover:text-[var(--color-text-main)]` replaces `text-gray-400 hover:text-white`
- **Tab separator:** `border-b border-[var(--color-cosmic-border)]` replaces `border-b border-gray-700`
- **Tab order:** Profile, Team, Security, API Keys, Webhooks, Cloudflare DNS (admin), Storage (admin), Restart Defaults (admin)
- **Admin-gated tabs:** Cloudflare DNS, Storage, Restart Defaults — only shown when `user.role === 'owner' || user.role === 'admin'`
- **No functional changes** — tab switching via `setActiveTab(id)` remains identical

### Glass Panel Section Wrapping
- Each tab's `<div className="space-y-6">` content wrapper gets `glass-panel p-6` applied at the tab container level (not per-subsection)
- Exception: Profile tab's Danger Zone confirmation (`bg-gray-800 rounded border border-red-900` on line 747) becomes `glass-panel border border-red-500/60 rounded-xl` with `bg-[var(--color-nebula)]` instead of `bg-gray-800`

### Profile Tab Sub-section Grouping (New — Structural Change)
The existing 6 flat subsections in `renderProfileTab` are regrouped under 4 sub-headings with visual dividers:

1. **Personal Information** — avatar upload, display name, profile info (name + email/change email)
   - Divider after: `pt-6 border-t border-[var(--color-cosmic-border)] mt-6`
   
2. **Account Security** — change password (moved from being between profile info and login history)
   - The existing password form at lines 596-637 stays in `renderProfileTab` but gets the heading "Account Security" instead of "Change Password"
   - Divider after: `pt-6 border-t border-[var(--color-cosmic-border)] mt-6`

3. **Login Activity** — login history table (keep expanded, no collapse)
   - Heading: "Login Activity" (replaces "Login History")
   - Divider after: `pt-6 border-t border-[var(--color-cosmic-border)] mt-6`

4. **Danger Zone** — delete account + transfer ownership
   - Keep red left border accent: `border-l-4 border-red-500/60 pl-4`
   - Confirmation panel: `glass-panel` with `border border-red-500/60`

- **No data model changes** — all handlers, state, and API calls stay untouched
- `getRoleBadge` helper function: stays in parent `SettingsPage.jsx` (shared by Profile and Team tabs)
