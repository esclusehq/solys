User-confirmed values (answered 2026-06-15):

| Role | Tailwind Class | Size | Weight | Line Height | Usage |

| Display | `text-3xl` | 30px | 600 semibold | 1.2 | Welcome heading (`getWelcomeMessage()`) |
| Heading | `text-2xl` | 24px | 600 semibold | 1.2 | Section titles: "Servers" / "Nodes" |

- `--font-mono: 'Fira Code', monospace` — used for code/terminal values (IP:Port, Node IDs, Domain)

## Interaction & Behavior Contract

### Loading State
- **Before** servers fetch resolves: show `SkeletonDashboard` (not full-screen `EscluseSpinner`)
- **Until both** servers AND nodes resolve: continue showing skeleton
- Combined loading: wait for `isLoading` (servers) AND `useNodes().loading` to be `false`
  - Implementation: introduce `const [pageLoading, setPageLoading] = useState(true)` → `setPageLoading(false)` after both resolve

### Empty States
- **No servers, no nodes:** Show both empty states independently (servers empty + nodes empty)
- **Has servers, no nodes:** Show servers table + nodes empty state
- **No servers, has nodes:** Show servers empty state + nodes table (edge case — unlikely but handled)
- Empty state containers: `glass-panel p-12 text-center` with CTA button + helpful links section below
- CTA buttons: `bg-[var(--color-cosmic-cyan)] text-[var(--color-deep-space)] px-6 py-3 rounded-lg font-semibold hover:brightness-110 transition-all`

### Tables — Search
- Dedicated `<input>` above each table section (between section heading and table container)
- Styling matches ServerManagerPage: `bg-[var(--color-cosmic-card)] border border-[var(--color-cosmic-border)] text-[var(--color-text-main)] px-4 py-2 rounded-lg w-full max-w-md`
- Filters `server.name` (servers) / `node.name` (nodes) — case-insensitive `.includes()`
- Debounce: 300ms (agent discretion — sensible default)
- No localStorage persistence for search text

### Tables — Status Filter
- `<select>` dropdown next to search input
- Styling matches search input: same cosmic style, `w-auto`
- Servers options: `All Status`, `Running`, `Stopped`, `Pending`
- Nodes options: `All Status`, `Online`, `Offline`
- No localStorage persistence for filter

### Tables — Sort
- `<select>` dropdown next to status filter
- Servers options: `Name A-Z`, `Running First`, `Last Activity`
- Nodes options: `Name A-Z`, `Online First`, `Uptime`
- Persist in localStorage per table: `dashboardServersSort`, `dashboardNodesSort` (matches Phase 74-75 pattern)
- Default sort: `Name A-Z` for both tables
- Implement sort function matching ServerManagerPage pattern (destructive copy of array before sort)

### Tables — Row Behavior
- View button only per row (D-10): no start/stop/restart actions
- View button styling: `px-3 py-1.5 text-xs rounded border border-[var(--color-cosmic-border)] text-[var(--color-text-muted)] hover:text-[var(--color-cosmic-cyan)] hover:border-[var(--color-cosmic-cyan)] transition-colors`
- Table row hover: `hover:bg-[rgba(255,255,255,0.02)] transition-colors`
- Row bottom border: `border-b border-[var(--color-cosmic-border)]`
- Table header: `text-left text-[var(--color-text-muted)] border-b border-[var(--color-cosmic-border)] bg-[rgba(255,255,255,0.02)]`

### Node Status Display (RESEARCH.md Gotcha #3)
- `StatusBadge` is server-centric (handles: running, crashed, degraded, starting, sleeping, stopped)
- Node statuses are simpler: `online` / `offline`
- Decision: Keep inline status badge for nodes with cosmic restyling (no `StatusBadge` reuse for nodes)
  - Online: `bg-[rgba(16,185,129,0.15)] text-[var(--color-cosmic-green)]` with `.status-dot.running` dot
  - Offline: `bg-[rgba(255,255,255,0.05)] text-[var(--color-text-muted)]` with gray dot
  - Match the pill shape of StatusBadge: `inline-flex items-center gap-2 px-3 py-1 rounded-full text-xs font-semibold`

### Billing Card Null Subscription
- Existing `getBillingInfo()` fallback preserved: `{ status: 'Free Plan', daysRemaining: '-', renewalDate: '-' }`
- Free Plan display: `text-2xl font-bold text-[var(--color-text-main)]` plus empty days left line (hide when `'-'`)

### Error Handling
- No inline error banners or retry buttons (user confirmed)
- Toast-only via `useUIStore().addToast({ type: 'error', message })`
- ToastContainer already global in App.jsx — no changes needed
- Error messages: descriptive of what failed + generic "please try again"

## Specific Interaction States

| State | Trigger | Visual | Behavior |
|-------|---------|--------|----------|
| Page loading | Initial mount | `SkeletonDashboard` renders (3 skeleton cards + server skeleton + nodes skeleton) | All data loading, no interactive elements |
| Servers loaded, nodes loading | After servers resolve, nodes still fetching | Keep skeleton for nodes section, servers table renders | Nodes section shows skeleton until its fetch resolves |
| Empty servers | `totalServers === 0` | `glass-panel p-12` with icon + heading + CTA button + helpful links | Clicking CTA navigates to `/servers` |
| Empty nodes | `totalNodes === 0` | Same structure as servers empty state | Clicking CTA navigates to `/nodes` |
| Table populated | `totalServers > 0` / `totalNodes > 0` | Cosmic-styled table with header + rows | Search/filter/sort controls visible above table |
| Search active | User types in search input | Live filtered rows (debounced 300ms) | Rows that don't match are hidden; "No servers match your filters" message if all hidden |
| Filter active | User selects filter dropdown | Dropdown shows options; table filtered | Only matching status rows shown |
| Sort changed | User selects sort option | Rows reorder immediately | Sort function applied to filtered array |
| Error loading servers | `serversApi.list()` throws | Toast notification bottom-right | Data remains as previous (or empty); no inline banner |
| Error loading nodes | `useNodes()` returns error | Toast notification | Data remains as previous |
| View button click | User clicks "View" on server row | Navigates to `/servers/{id}` | Standard `<Link>` or `navigate()` |
| View button click (node) | User clicks "View" on node row | Navigates to `/nodes` | No per-node detail page exists yet |

### Specific: "No results match your filters" State
When search/filter active but no rows match:
```
<div className="text-[var(--color-text-muted)] text-center py-8">
  No servers match your filters.
</div>
```
This is distinct from the empty state (which shows CTA + helpful links).
