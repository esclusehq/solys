## Spacing Scale

Using Tailwind v4 default spacing scale (multiples of 4). Inherited from Phase 75 without changes:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, compact inline padding (e.g. `gap-1`, `p-1`) |
| sm | 8px | Compact element spacing, badge padding (e.g. `gap-2`, `p-2`) |
| md | 16px | Default element spacing, button padding (e.g. `gap-4`, `p-4`, `px-4 py-2`) |
| lg | 24px | Section padding, card internal padding (e.g. `p-6`, `gap-6`) |
| xl | 32px | Layout gaps between sections (e.g. `gap-8`, `p-8`) |
| 2xl | 48px | Major section breaks, page header margins (e.g. `mb-12`) |
| 3xl | 64px | Page-level outer spacing (e.g. `p-8` wrapper, `gap-16`) |

Exceptions: **none** — all billing UI conforms to existing Tailwind 4px grid.

## Interaction Contract

### Section Layout (D-01)

```
Page heading "Billing" (h1, text-2xl, font-bold, mb-8)
↓
┌─────────────────────────────────────────────┐
│  Subscription Status (glass-panel, p-6, mb-8) │
│  Current plan name + status badge            │
│  Usage bars (servers, RAM, CPU, disk)        │
│  Nodes (text-only limit display)             │
│  [Berhenti Berlangganan] button (destructive) │
└─────────────────────────────────────────────┘
                 ↓
            border-b separator
                 ↓
┌─────────────────────────────────────────────┐
│  Refund Eligibility (glass-panel, p-6, mb-8)  │
│  (shown only when subscription + eligibility) │
└─────────────────────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────┐
│  Refund History (glass-panel, p-6, mb-8)      │
│  (shown only when refundHistory.length > 0)   │
└─────────────────────────────────────────────┘
                 ↓
        2-column grid (md:grid-cols-2 gap-6)
    ┌─────────────────┐ ┌─────────────────┐
    │  Hobby Plan Card  │ │  Pro Plan Card   │
    │  (glass-panel)    │ │  (glass-panel)    │
    └─────────────────┘ └─────────────────┘
                 ↓
┌─────────────────────────────────────────────┐
│  Payment History (glass-panel, p-6)           │
│  (shown only when invoices.length > 0)        │
└─────────────────────────────────────────────┘
```

**Section order (locked per CONTEXT.md):** Subscription Status → Refund (conditional) → Plan Cards (2-column) → Payment History

**Visual separator:** `border-b border-[var(--color-cosmic-border)]` between sections — applied to the bottom of each glass-panel or as a separate `<hr>`.

### Subscription Status Card (D-02)
- **Container:** `glass-panel p-6` with `border border-[var(--color-cosmic-cyan)]` accent when subscription is active
- **Header:** "Current Plan: {name}" (text-xl, font-bold) with status badge inline
- **Status badge:** `bg-[var(--color-cosmic-green)]/10 text-[var(--color-cosmic-green)]` for active, `bg-[var(--color-cosmic-orange)]/10 text-[var(--color-cosmic-orange)]` for non-active (matching Nodes.jsx pattern)
- **Limits grid:** `grid grid-cols-2 md:grid-cols-5 gap-4` — labels in `text-text-muted text-sm`, values in `text-white font-bold`
- **No subscription (Free tier):** Same layout, border becomes `border-[var(--color-cosmic-border)]`, shows "Current Plan: Free" with Status: none

### Usage Bars — New Feature (D-03)

**Layout:** Below limits text, inside the subscription card:
- Section heading: "Usage" (text-sm, uppercase, tracking-wide, `text-text-muted`, mb-3)
- One bar per resource: Servers, RAM (MB), CPU Cores, Disk (GB)
- Nodes: text-only — "Max Nodes: {value}" label, no bar (per CONTEXT.md decision)

**Per-bar structure (left to right):**
```
Resource name (label, text-sm, text-text-muted)
Current / Max (number, text-sm, text-white font-medium, text-right)
[████████░░░░░░░░░░░░]  <-- progress bar
```

**Progress bar visual:**
```jsx
<div className="h-2 rounded-full bg-gray-700 overflow-hidden">
  <div
    className="h-full rounded-full transition-all duration-300"
    style={{
      width: `${percentage}%`,
      backgroundColor: getBarColor(percentage) // dynamic
    }}
  />
</div>
```

**Color thresholds** (locked per CONTEXT.md):
| Usage | Threshold | Color |
|-------|-----------|-------|
| Green | < 60% | `var(--color-cosmic-green)` #10b981 |
| Yellow | 60-85% | `var(--color-cosmic-orange)` #f59e0b |
| Red | > 85% | `var(--color-cosmic-red)` #ef4444 |

**Special cases:**
- **Max = -1 ("Unlimited"):** Show full bar at 100% with `bg-gray-600` (muted, not a color threshold). Label shows "Unlimited" instead of number. No fill movement.
- **Current = 0 and max != -1:** Bar at 0%, show "0 / {max}".
- **Percentage > max (overage):** Cap at 100%, use red color, show actual numbers (e.g., "6 / 5").

### Plan Cards (D-04)
- **Container:** `glass-panel p-6` with `border border-[var(--color-cosmic-border)]` default
- **Hover:** `hover:border-[var(--color-cosmic-cyan)]/40 transition-all` glow effect (matching PluginManager pattern from CONTEXT.md)
- **Layout (top to bottom, flex flex-col h-full):**
  1. Emoji icon (text-3xl) + plan name (text-2xl font-bold) + price (text-3xl font-bold)
  2. Description (text-text-muted mt-1)
  3. Limits row: "Max Servers: {value}" | "Max Nodes: {value}" (text-sm)
  4. Features list with green checkmarks (✓)
  5. "Best for: {value}" (text-xs italic text-text-muted)
  6. CTA button or Current Plan badge (mt-auto, sticky bottom)
- **CTA button:** Full width, `bg-[var(--color-cosmic-cyan)] hover:opacity-90 text-white py-2.5 rounded font-medium transition-all`
- **Current Plan badge:** Full width, `bg-[var(--color-cosmic-green)]/20 text-[var(--color-cosmic-green)] py-2.5 rounded text-center font-medium text-sm` — non-interactive, shows "Current Plan"

### Berhenti Berlangganan Button (D-05)
- **Visual:** `bg-[var(--color-cosmic-red)] hover:bg-[var(--color-cosmic-red)]/80 text-white px-4 py-2 rounded font-medium transition-colors` — destructive red button, placed below limits grid
- **Interaction:** Click → browser `window.confirm()` → on confirm → `billingApi.cancelSubscription()` → success toast "Subscription cancelled successfully" → `loadData()`
- **Kept as-is** per CONTEXT.md (no upgrade to modal)
- **Position:** Bottom of subscription card, after limits/usage section, left-aligned

### Request Refund Button (D-06)
- **Visual:** Same destructive red style as Berhenti Berlangganan
- **Interaction:** Click → `billingApi.requestRefund()` directly (no confirmation dialog) → success toast "Refund request submitted" → `loadData()`
- **Conditionally shown:** Only when `refundEligibility.eligibility !== 'none'`

### Payment History Table (D-07)
- **Container:** `glass-panel p-6 overflow-x-auto`
- **Table:** Full width, `border-collapse`
- **Header row:** `text-text-muted text-left border-b border-[var(--color-cosmic-border)]`, columns: Date | Amount | Status
- **Body rows:** `border-b border-[var(--color-cosmic-border)]`, text-white for values
- **Status badges:**
  - `paid`: `bg-[var(--color-cosmic-green)]/20 text-[var(--color-cosmic-green)]`
  - `pending` / `open`: `bg-[var(--color-cosmic-orange)]/20 text-[var(--color-cosmic-orange)]`
  - `past_due` / `failed`: `bg-[var(--color-cosmic-red)]/20 text-[var(--color-cosmic-red)]`

### Refund Eligibility (D-08)
- **Kept as-is** per CONTEXT.md (emoji indicators 🟢🟡🔴 retained)
- **Container:** `glass-panel p-6` with `border border-[var(--color-cosmic-orange)]/30` (yellow-tinted to match refund context)
- **Layout:** Emoji (text-2xl) + status text (font-bold) + description (text-sm text-text-muted) in a horizontal flex row
- **CTA:** Red destructive button "Request Refund" (only shown when eligibility is `full` or `prorated`)

### Loading State (D-09)
- **Visual:** `EscluseSpinner` centered in page (existing component, size=100, color="#06b6d4")
- **Container:** `min-h-screen flex items-center justify-center`
- **Behavior:** Shows during initial `loadData()` Promise.all — hidden once data resolves or errors
- **Unchanged** from current implementation per CONTEXT.md

### Data Loading Pattern (D-10)
- **Parallel fetch:** `Promise.all([billingApi.getPlans(), billingApi.getCurrentSubscription(), billingApi.getInvoices(), usageApi.getQuotas()])` in `loadData()`
- **Waterfall:** Refund eligibility + refund history fetched after subscription resolves (only if active subscription exists)
- **Error isolation:** If refund fetch fails → logged to console, section hidden; if billing fails → toast error, null state shown; if usage fetch fails → usage section hidden or shows dashes
- **Usage API:** New `usageApi.getQuotas()` endpoint → `api.get('/usage/quotas')` — returns `{ plan, limits, current_usage }`
