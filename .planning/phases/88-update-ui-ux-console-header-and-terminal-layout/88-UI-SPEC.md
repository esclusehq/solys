## Design System

| Property | Value |

## Spacing Scale

All values in pixels. Standard 4px base scale from Tailwind CSS v4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, toolbar button inner padding, status dot margins |
| sm | 8px | Compact element spacing, sidebar item gap |
| md | 16px | Default element spacing, terminal title bar padding |
| lg | 24px | Section padding, terminal container padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

**Exceptions:**

| Element | Value | Rationale |
|---------|-------|-----------|
| Floating toolbar buttons | 6px (`p-1.5`) | Compact toolbar — 24px touch target per WCAG with 14px icon |
| Sidebar toggle button | 24px (`w-6 h-6`) | Small circular toggle peg, non-critical action |
| Status dots | 8px (`w-2 h-2`) | Existing status-dot pattern; no change |
| Sidebar width (open) | 224px (`w-56`) | Matches server sidebar content (name + dot); narrower than 240px main sidebar |
| Sidebar width (collapsed) | 0px with `overflow-hidden` | Collapses completely; toggle button positioned absolutely |
| Terminal title bar | 12px height, 8px horizontal padding | Slim single-line status bar |

**Source:** D-01 through D-14 (CONTEXT.md), existing `Terminal.jsx`, `Sidebar.jsx` patterns, Tailwind default spacing.

### Theme Adaptability

All colors are defined via CSS variables that switch between dark and light themes via the `[data-theme="light"]` selector in `index.css`. Components use variable references (e.g., `var(--color-cosmic-cyan)`) — never hardcoded hex values.

**Exception — xterm.js theme:** The terminal canvas uses the existing hardcoded cosmic theme object (D-05). This should be extracted to `src/theme/terminalTheme.js` for consistency but remains a JS config object (CSS variables don't apply inside xterm.js canvas).

### Accent Reserved For

Cyan accent (`--color-cosmic-cyan`) is reserved exclusively for:

| Element | Specific Usage |
|---------|----------------|
| Floating toolbar buttons | Hover text color, hover border when Reconnect is shown |
| Overlay "Retry Now" button | bg/10, text, border/30, hover:bg/20 |

### Console Page Layout (`Console.jsx`)

- Flex column container using existing pattern: `flex-1 flex flex-col overflow-hidden h-full min-h-0`
- **Header row:** `shrink-0` — breadcrumb "← Back to Server" (left) + server name display (center, replaces "Console" title)
- **Body area:** `flex-1 flex overflow-hidden` — sidebar + terminal side by side
- **No max-width container** — terminal should use full available width

### Server Sidebar (`ServerSidebar.jsx` — NEW)

- **Position:** Left side of body, before terminal container
- **States:** Open (224px) | Collapsed (0px, hidden)
- **Default:** Collapsed (D-08)
- **Toggle:** Absolute-positioned button (~16px circle) pegged to the right edge of sidebar, vertically aligned with the top of the terminal
- **Toggle icon:** `PanelLeft` (lucide-react) when collapsed, `X` when open
- **Animation:** CSS transition `width 300ms ease` on the sidebar container; `overflow-hidden` to clip content during transition
- **Content (open):** 12px padding wrapper with "SERVERS" header label + scrollable list
- **Content (collapsed):** Nothing visible; toggle button remains accessible
- **Server list styling:** Each entry = flex row with 8px status dot + truncated name. Active server highlighted with subtle background tint
- **Scroll:** Vertical scroll if server list exceeds available height — use existing `::-webkit-scrollbar` styles from `index.css`

### Terminal Area

- **Container:** `flex-1 flex flex-col overflow-hidden` — fills remaining space after sidebar
- **Flex direction:** column (title bar + canvas)
- **Title bar:** `shrink-0` single row at top — status badge (dot + text) on the left, Clear button on the right
- **Canvas container:** `flex-1 relative overflow-hidden` — houses xterm.js element, floating toolbar, disconnect overlay, search bar
- **xterm.js element:** Fills container via FitAddon; no external padding on the xterm `<div>` (use `p-0` instead of current `p-3`)

### Floating Toolbar

- **Position:** `absolute top-2 right-2 z-10`
- **Pointer events:** Container = `pointer-events: none`; individual buttons = `pointer-events: auto`
- **Layout:** Horizontal flex row with `gap-1` between buttons
- **Default opacity:** 0.3
- **Hover opacity:** 1.0 (transition `opacity 200ms ease`)
- **Visibility trigger:** On hover over terminal top-right area; 2s fade-out timer after mouse leaves
- **Z-index:** Above xterm canvas (z-10), below disconnect overlay (z-20), below search bar (z-10, peers with toolbar)

### Disconnect Overlay

- **Position:** `absolute inset-0 z-20` over the canvas container
- **Backdrop:** `absolute inset-0 bg-[var(--color-deep-space)]/60 backdrop-blur-[2px]` — translucent, terminal output remains visible
- **Content:** Centered flex column with vertical spacing:
  1. "Disconnected" heading (red, semibold)
  2. "Reconnecting in Xs... (attempt N/10)" countdown (muted text)
  3. "Retry Now" button
- **Visibility:** Shown only when disconnected (connected=false AND reconnectAttempts > 0)
- **Button click:** Calls manual reconnect, hides overlay
- **Pointer events:** Backdrop = `pointer-events: none`; content area/button = `pointer-events: auto`

### Search Bar (Ctrl+F)

- **Position:** `absolute top-10 right-2 z-10` — slides down below toolbar
- **Container:** Glass-panel styled: bg-cosmic-card, border-cosmic-border, rounded-lg, backdrop-blur
- **Layout:** Horizontal flex row: input field + match count label + prev/next buttons + close button
- **Visibility:** Toggled by Search button in toolbar or Ctrl+F keybinding
- **Auto-focus:** Input receives focus when opened

### Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| > 1024px | Full layout: sidebar + terminal + toolbar |
| 768–1024px | Sidebar auto-collapses by default, toggle still available |
| < 768px | Sidebar forced collapsed, floating toolbar shrinks (icon-only, no labels) |

**Source:** D-01 through D-14 (CONTEXT.md), `Console.jsx` existing layout, `Terminal.jsx` existing container structure, RESEARCH.md floating toolbar and sidebar patterns.

## Interaction Contracts

### 1. Inline Terminal Input (D-03)

| Aspect | Specification |
|--------|--------------|
| **Input mechanism** | `term.onData()` handler — keystrokes written to xterm.js buffer and accumulated in `useRef<string>` |
| **Buffer management** | `useRef` NOT `useState` — avoids re-render on every keystroke |
| **History storage** | `useRef` array (not useState) + `localStorage` for persistence |
| **Prompt format** | `"> "` displayed after each command completion and on new terminal connection |
| **Enter** | Commit input buffer: save to history, echo command with newline, send via WebSocket `JSON.stringify({type:'command', command})`, reset buffer, write prompt |
| **Arrow Up** | Navigate backward through history. Rewrite current line with `\r\x1b[K` + prompt + history entry |
| **Arrow Down** | Navigate forward through history. Rewrite to next entry or clear to empty prompt |
| **Backspace** | Remove last char from buffer, send `\b \b` sequence to terminal |
| **Ctrl+C** | Send `^C`, cancel current input, reset buffer, write new prompt |
| **Tab** | Autocomplete from known commands list (`commonCommands` array, existing in Terminal.jsx) |
| **Printable chars** | Append to buffer, write char to terminal |
| **Command formats sent** | `{type: 'command', command: '<buffer>'}` via WebSocket |
| **Disable input** | When WS not connected, ignore onData except for cursor nav arrows |
| **Edge case — long lines** | Cap buffer at terminal `cols` to prevent horizontal overflow; truncate silently |
| **Edge case — Ctrl+Z / escape sequences** | Pass unrecognized non-printable sequences through silently (no-op) |

### 2. Floating Toolbar (D-02)

| Aspect | Specification |
|--------|--------------|
| **Default visibility** | `opacity-30` — subtle, non-distracting |
| **Hover activation** | Mouse enters toolbar area → full `opacity-100`; 2s timeout on mouse leave before fade back |
| **Transition** | `opacity 200ms ease` (fast fade, not abrupt) |
| **Pointer event layer** | Container: `pointer-events: none`; buttons: `pointer-events: auto` |
| **Button: Clear** | Click → `term.clear()`, clear input buffer |
| **Button: Reconnect** | Click → close existing WS, reset reconnectAttempts to 0, call `connect()`. Only shown when disconnected (red-tinted) |
| **Button: Search** | Click → toggle search bar visibility. Ctrl+F keyboard shortcut also toggles |
| **Button: Copy** | Click → copy terminal selection to clipboard via `navigator.clipboard.writeText(term.getSelection())` |
| **Button: Help ("?")** | (Optional) Click → popover showing keyboard shortcuts: Enter=Send, ↑↓=History, Tab=Autocomplete, Ctrl+C=Cancel, Ctrl+L=Clear |
| **Tooltip on each button** | Show on hover after 500ms delay. Uses existing tooltip pattern (if available) or native `title` attribute |
| **Edge case — disconnected** | Reconnect button pulses or glows red to draw attention; other buttons remain at default opacity |

### 3. Server Sidebar (D-06 through D-09)

| Aspect | Specification |
|--------|--------------|
| **Toggle position** | Absolute circle button at `-right-3 top-4` of sidebar container, `z-20` |
| **Toggle icon** | `PanelLeft` → `X` transition (or `ChevronRight` → `ChevronDown`) |
| **Open animation** | Width transitions from 0 → 224px over `300ms ease`; content fades in |
| **Close animation** | Width transitions from 224px → 0 over `300ms ease`; content fades out |
| **Servers data source** | `useServers()` hook lifted to Console.jsx, passed as prop to ServerSidebar (prevents duplicate fetches) |
| **Server selection** | Click on server entry → `setSelectedId(server.id)` — triggers new WS connection |
| **Active server highlight** | Current `selectedId` entry gets `bg-[rgba(255,255,255,0.08)] text-[var(--color-text-main)]` |
| **Status dot color** | `running` → green (`--color-cosmic-green`) with glow; `stopped` → red; other/null → orange |
| **Sidebar label** | "SERVERS" uppercase semibold 12px label at top of open sidebar |
| **Server name visible when collapsed** | D-09: In the ConsoleHeader component, show the active server name (e.g., "My Minecraft Server") replacing the "Console" page title |

### 4. Connection Status (D-10 through D-13)

| Aspect | Specification |
|--------|--------------|
| **Location** | Terminal title bar (top of terminal container, not in header, not in sidebar) |
| **Format** | Status dot (8px) + label text, inline flex, positioned left in title bar |
| **Connected state** | Green dot (`bg-[var(--color-cosmic-green)] shadow-[0_0_6px_var(--color-cosmic-green)]`) + "Connected" text in green badge |
| **Disconnected state** | Red dot (`bg-[var(--color-cosmic-red)]`) + "Disconnected" text in red badge |
| **Connecting state** | Orange dot + "Connecting..." text (brief transitional state) |
| **Reconnect overlay shown when** | `connected === false` AND `reconnectAttempts > 0` |
| **Overlay copy** | "Disconnected" (heading, red) + "Reconnecting in Xs... (attempt N/10)" (muted text) |
| **Overlay countdown** | Derived from exponential backoff delay: `min(1000 * 2^attempt, 30000) / 1000` rounded down |
| **Overlay "Retry Now" button** | Click → `clearTimeout(reconnectTimeout)`, `reconnectAttempts = 0`, `connect()`, hide overlay |
| **Toolbar Reconnect button** | Always visible when disconnected; immediate retry, resets backoff counter |
| **Edge case — max attempts** | After 10 failed attempts, stop auto-reconnect. Overlay shows "Disconnected — Max reconnection attempts reached. [Retry Now]" |
| **Edge case — manual reconnect** | User clicks Retry Now or toolbar Reconnect → reset attempts to 0, connect immediately |

### 5. Search (Ctrl+F, via @xterm/addon-search)

| Aspect | Specification |
|--------|--------------|
| **Library** | `@xterm/addon-search` 0.16.0 |
| **Toggle** | Ctrl+F or click Search icon in toolbar |
| **Search bar position** | Absolute, below toolbar (top-10 right-2), glass-panel styled |
| **Input** | Auto-focused `<input>` with placeholder "Find in terminal..." |
| **Navigation** | Up (↑) / Down (↓) arrow buttons for `findPrevious()` / `findNext()` |
| **Incremental search** | On each keystroke, call `searchAddon.findNext(term, { incremental: true })` |
| **Decorations** | `matchOverviewRuler: '#0ddff2'`, `matchBackground: '#0ddff220'`, `activeMatchBackground: '#0ddff240'` |
| **Close** | `X` button, Escape key, or click Search icon again |
| **Match count** | Displayed in search bar as "N matches" — derived from addon API or tracked manually |
| **No results** | Show "No matches" text; input border glows red briefly |
| **State reset** | On close: `searchAddon.clearDecorations()`, clear input |

### 6. Console Header (D-14)

| Aspect | Specification |

| **Server name source** | Derived from `servers.find(s => s.id === selectedId)?.name` or `selectedServerName` | 
| **When no server selected** | Header shows "← Back" link hidden, Title shows "Console" |
