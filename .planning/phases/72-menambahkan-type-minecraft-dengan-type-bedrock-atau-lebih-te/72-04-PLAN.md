  <files>

  <read_first>
    @.planning/phases/72-menambahkan-type-minecraft-dengan-type-bedrock-atau-lebih-te/72-RESEARCH.md (Pitfall 5 — Relay Tunnel TCP Assumption)
  </read_first>
  <action>
    Document the following known limitations for Phase 72 Bedrock MVP:

    **Relay Tunnel (Pitfall 5):**
    The relay infrastructure (Phase 68) uses yamux over WebSocket (TCP). Minecraft Bedrock uses UDP/RakNet protocol. The relay gateway does NOT support UDP forwarding. Therefore:
    - Bedrock servers operate in **Direct Mode only** — no relay failover
    - The `relay_service.rs` does not push relay config for bedrock servers (they default to direct mode via existing behavior since relay is only set up when connectivity mode override is explicitly set)
    - This is acceptable for MVP — Bedrock servers work without relay
    
    **Console/Terminal (Open Question 2):**
    Bedrock has no RCON support. The itzg/minecraft-bedrock-server image supports stdin-based console via `docker exec -i`. Console support for Bedrock is deferred — not implemented in Phase 72.
    
    **Behavior Packs / Addons (Open Question 3):**
    Bedrock supports `.mcpack`, `.mcworld`, `.mcaddon` files. Deferred — not implemented in Phase 72.

    **Decision needed:** Are these limitations acceptable for the Bedrock MVP? If yes, proceed. If no, scope must be expanded.
  </action>
  <verify>
    <automated>N/A — decision checkpoint</automated>
  </verify>
  <decision>
    Bedrock MVP known limitations:
    1. **No Relay support:** Bedrock servers use Direct Mode only (TCP relay cannot forward UDP)
    2. **No Console:** Bedrock has no RCON; console/terminal not implemented (use docker exec)
    3. **No Addons/Behavior Packs:** Not implemented; can be configured via env vars manually
    4. **No Version Selector:** Bedrock always uses `LATEST` from Docker image (no version dropdown in UI)
  </decision>
  <context>
    These limitations are explicitly documented as "out of scope for Phase 72" in RESEARCH.md. The MVP goal is to make Bedrock servers deployable and runnable. Relay support and advanced features are future enhancements.
