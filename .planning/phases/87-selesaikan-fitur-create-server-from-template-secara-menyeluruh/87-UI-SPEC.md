## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (p-1) |
| sm | 8px | Compact element spacing, badge padding (p-2) |
| md | 16px | Default card padding, form field gaps (p-4) |
| lg | 24px | Section padding within pages (p-6) |
| xl | 32px | Layout gaps, modal padding (p-8) |
| 2xl | 48px | Major section breaks (my-12, py-12) |
| 3xl | 64px | Page-level spacing (p-16) |

Exceptions:
- Icon-only touch targets: min 44px per WCAG (use p-3 on icon buttons)
- `status-dot`: 8px circle (not a 4px multiple but intentional per existing CSS)

**Source:** Pre-populated from existing Tailwind v4 default scale + cosmic theme patterns. Confirmed by `app/src/index.css` and existing component analysis (TemplateCard, CreateServerModal, TemplateLibraryPage).

## Component Inventory & Interaction Spec

### New Components Required

#### 1. TemplateDetailPage (`app/src/pages/templates/TemplateDetailPage.jsx`)
- **Route:** `/templates/:id`
- **States:**
  - **Loading:** Full-page `EscluseSpinner` centering (matching TemplateLibraryPage pattern)
  - **Error:** Inline `glass-panel` alert with `--color-cosmic-red` border, error message, Retry button
  - **Not found:** Empty state with "Template not found" heading + body copy
  - **Loaded (active):** Full detail layout (see Layout Structure below)
  - **Loaded (inactive — Coming Soon):** Same layout but with "Coming Soon" badge + disabled Create button
- **Layout structure** (glass-panel, max-w-6xl mx-auto, p-8):
  ```
  ┌─────────────────────────────────────────┐
  │  ← Back to Templates (breadcrumb link)  │
  │                                         │
  │  Header row:                            │
  │  [Template Name]        [v2.1] [Official]│
  │  Minecraft Java / Paper                  │
  │                                         │
  │  Description (full text)                 │
  │                                         │
  │  Metadata row:                           │
  │  📅 Last updated X ago    👥 X servers   │
  │                                         │
  │  ┌─ Config Preview ──────────────────┐  │
  │  │  RAM: 4 GB    Disk: 20 GB          │  │
  │  │  Game Type: Minecraft Java          │  │
  │  │  Version: 26.2    Players: 20       │  │
  │  └───────────────────────────────────┘  │
  │                                         │
  │  ┌─ Plugins & Mods (if any) ────────┐  │
  │  │  • EssentialClient v1.3 (1.21+)    │  │
  │  │  • WorldEdit v7.3 (1.21+)          │  │
  │  └───────────────────────────────────┘  │

  │  [Create Server] ←─── primary CTA       │
  │                                         │
  │  Resource Usage:                        │
  │  ┌─ Node Capacity Reference ─────────┐  │
  ```

- **Interactions:**
  - "← Back to Templates" links to `/templates`
  - "Create Server" button opens ConfigureServerModal (see below)
  - Breadcrumb uses text-sm text-muted, hover:text-primary
- **Data fetching:** `templatesApi.get(id)` on mount; loading/error states handled

#### 2. ConfigureServerModal (`app/src/features/templates/ConfigureServerModal.jsx`)
- **Trigger:** Opened by "Create Server" button on TemplateDetailPage
- **States:**
  - **Open:** Modal overlay with backdrop blur, centered glass-panel container
  - **Loading (nodes):** Node dropdown shows "(Loading...)" suffix, disabled
  - **Error (nodes):** Toast notification, dropdown shows "Auto-select best node" only
  - **Validating:** Real-time port validation on input change
  - **Submitting:** Button shows "Deploying...", all form fields disabled, loading spinner on button
  - **Success:** Toast "Server created successfully!", modal closes, navigate to `/servers/{id}`
  - **Error submit:** Toast "Failed to create server: {message}", fields re-enabled
- **Layout structure** (single scrollable form within glass-panel, max-w-lg, max-h-[90vh] overflow-y-auto):
  ```
  ┌─────────────────────────────────────┐
  │ [X]                                  │
  │ Configure Server   Step 1 of 2       │
  │ ████████████░░░░░░ 50%               │
  │                                      │
  │ ─── Basic Config ───               │
  │ Server Name: [________________]     │
  │ Target Node: [Auto-select best ___▼]│
  │ Online Mode: [On ●━━━━━━━○ Off ]    │
  │                                      │
  │ ─── Resources ───                   │
  │ RAM: [-] ████████░░░ 4 GB [+]      │
  │ Disk: [-] ██████░░░░░ 20 GB [+]    │
  │ ⚠️ Node capacity warning (if node   │
  │    selected and insufficient RAM)    │
  │                                      │
  │ ─── Game Settings ───              │
  │ World Seed: [Leave empty for______] │
  │ Max Players: [20 ▼]                 │
  │                                      │
  │ ─── Dependencies (2) ───           │
  │ ✅ EssentialClient v1.3             │
  │ ✅ WorldEdit v7.3                   │
  │                                      │
  │ [Cancel]         [Deploy Server]     │
  └─────────────────────────────────────┘
  ```
- **Form fields (all pre-filled from template defaults per D-05):**
  | Field | Type | Default | Notes |
  |-------|------|---------|-------|
  | Server Name | text input | template default or "" | required, max 64 chars |
  | Target Node | dropdown | "Auto-select best node" | Options: user's registered nodes; loading/empty states |
  | RAM | stepper (± buttons) | template default (GB) | Min: 2, Max: 32, Step: 2, from `RESOURCE_LIMITS.ram` |
  | DISK | stepper (± buttons) | template default (GB) | Min: 10, Max: 200, Step: 10, from `RESOURCE_LIMITS.disk` |
  | Online Mode | toggle switch | template default | On/Off, use existing `true`/`false` select or custom toggle |
  | World Seed | text input | empty (optional) | placeholder: "Leave empty for random" |
  | Max Players | dropdown | template default | Options: 10, 20, 30, 50, 100 |
  - **Locked fields** (displayed as read-only, not editable per D-06): Game Type, Version, Category
- **Resource warning (D-07, D-08, D-09):**
  - When a node is selected AND template RAM > node available RAM:
    - Show warning in a `--color-cosmic-orange`/20 bg panel with orange border
    - Text: "⚠️ **{Node name}** has only **{X} GB RAM** available. This template recommends **{Y} GB RAM**. You can still proceed with reduced resources."
  - Warning is non-blocking — user can still submit (D-08)
  - If no node selected, skip warning (D-09 — references agent node, not billing plan)
- **Dependency preview (D-10, D-11):**
  - Section at bottom of modal showing all plugins/modpacks that will be auto-installed
  - Each item shows: plugin/mod name, version, game version compatibility
  - If no dependencies: "No additional plugins or mods required — this template works out of the box."
  - User must scroll through to confirm (no separate confirmation checkbox needed — submission is implicit confirmation)
- **Form validation:**
  - Server Name: required, non-empty
  - RAM: integer, between min/max
  - DISK: integer, between min/max
  - World Seed: optional, no validation
  - Max Players: integer from dropdown

### Stepper Component Pattern (RAM, DISK)
Reuse the existing stepper pattern from `ServerOnboardingWizard.jsx` (lines 130-158):
```
[-] ████████░░░░ 4 GB [+]
```

- Progress bar: `h-2 bg-[var(--color-cosmic-border)] rounded-full` with `bg-[var(--color-cosmic-cyan)]` fill
- Width calculated as `((val - min) / (max - min)) * 100`%
- Buttons disabled at min/max with `disabled:opacity-30 disabled:cursor-not-allowed`

## State Machine Summary

### TemplateDetailPage
| State | Trigger | Visual | User Action |
|-------|---------|--------|-------------|
| Loading | Mount component | Full-page EscluseSpinner | None (wait) |
| Error | API failure | Inline red alert + Retry | Click Retry |
| Not found | 404 response | Empty state card | Navigate away |
| Loaded (active) | API success | Full detail layout | Click "Create Server" |
| Loaded (inactive) | API success + !is_active | Same layout, button disabled | Navigate away or back |

### ConfigureServerModal
| State | Trigger | Visual | User Action |
|-------|---------|--------|-------------|
| Open | "Create Server" click | Modal with pre-filled form | Edit fields |
| Nodes loading | Modal open | "(Loading...)" in dropdown | Wait |
| Validation error | Invalid input | Inline error text | Fix input |
| Submitting | "Deploy Server" click | Button disabled "Deploying...", fields disabled | Wait |
| Success | API 200 | Toast + auto-close + redirect | None (automatic) |
| Error | API failure | Toast with error message | Retry or cancel |

## Accessibility

- All form inputs must have associated `<label>` elements
- Modal: `role="dialog"`, `aria-modal="true"`, backdrop click closes
- Focus trap inside modal (Tab cycles through modal elements only)
- Escape key closes modal (match existing CreateServerModal pattern)
- Toggle switches: `role="switch"`, `aria-checked`
- Stepper buttons: `aria-label="Decrease RAM"` / `aria-label="Increase RAM"`
- Loading states: `aria-busy="true"` on form during submission
- Error messages: `role="alert"` on inline validation errors
- Color contrast: All text on cosmic backgrounds must pass WCAG AA (4.5:1)
  - `--color-text-main` (#e2e8f0 on #080b15) = ~13.7:1 ✓
  - `--color-text-muted` (#64748b on #080b15) = ~6.3:1 ✓
  - `--color-cosmic-cyan` (#0ddff2 on #080b15) = ~9.2:1 ✓
