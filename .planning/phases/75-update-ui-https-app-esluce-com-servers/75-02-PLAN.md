    **Step 3: Add polling useEffect** — add after the existing initial fetch useEffect (the one with `fetchServers().finally(...)`). Insert this block after its closing bracket:
    ```javascript
    // 30s polling with status change detection (D-09, D-10)
    useEffect(() => {
      const interval = setInterval(async () => {
        const oldServers = prevServersRef.current
        await fetchServers()
        const newServers = useServerStore.getState().servers
        // Detect status changes between polls
        newServers.forEach(newS => {
          const oldS = oldServers.find(s => s.id === newS.id)
          if (oldS && oldS.status !== newS.status) {
            addToast({ type: 'info', message: `${newS.name} is now ${newS.status}` })
          }
        })
        prevServersRef.current = newServers
      }, 30000)

      return () => clearInterval(interval)
    }, []) // eslint-disable-line react-hooks/exhaustive-deps
    ```

    **Step 4: Initialize prevServersRef after initial fetch** — add a second useEffect that populates the ref after the first fetch completes. Insert after the polling useEffect:
    ```javascript
    // Initialize prevServersRef when servers first load
    useEffect(() => {
      if (servers.length > 0 && prevServersRef.current.length === 0) {
        prevServersRef.current = servers
      }
    }, [servers])
    ```

    KEY DESIGN DECISIONS (per CONTEXT.md D-09, D-10, UI-SPEC Interaction Contract):
    - No polling indicator in the UI — refetch is silent per UI-SPEC
    - Status change fires for ALL transitions, including starting→running (per RESEARCH.md Open Q2 recommendation)
    - Uses `useServerStore.getState().servers` inside the interval, NOT the closure variable `servers`, to always get the latest state
    - `useRef` stores previous snapshot — no `useEffect` dependency on `servers` for change detection (avoids Pitfall 2: infinite loop)
    - Cleanup function calls `clearInterval(interval)` (avoids Pitfall 1: memory leak)
    - Initial fetch already happens in the existing useEffect (line ~15-17), so no duplicate fetch on mount
