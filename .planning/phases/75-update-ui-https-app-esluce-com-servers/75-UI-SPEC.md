## Interaction Contract

### View Toggle (D-02, D-03)
- **Visual:** Segmented button group with two icon buttons: LayoutGrid icon (card view) and List icon (table view). Active state uses `var(--color-cosmic-cyan)` text/border. Inactive uses `var(--color-text-muted)`.
- **Accessibility:** Each toggle button uses `aria-label` — `aria-label="Card view"` and `aria-label="Table view"`.
- **Persistence:** View preference saved to `localStorage` key `serverViewMode` with values `'card'` or `'table'`.
- **Default:** `'card'` (current default view).
- **Placement:** Right side of the filter bar, below the page header.
- **Focal point:** The server card/table grid is the primary visual anchor. The + Add Server button is the primary action CTA in the top-right.

### Sort Controls (D-04, D-06)
- **Visual:** Dropdown select styled like the existing filter selects in `ServerManagerPage.jsx` — same `bg-gray-800 border border-gray-700 text-white px-4 py-2 rounded-lg` pattern, ported to cosmic variables: `bg-[var(--color-cosmic-card)] border border-[var(--color-cosmic-border)]`.
- **Options:** "Sort: Name A-Z" (default), "Sort: Running First", "Sort: Last Activity".
- **Persistence:** Sort preference saved to `localStorage` key `serverSortMode`.

### Game Type Filter (D-05)
- **Visual:** Dropdown select styled identically to the status filter.
- **Options:** "All Games" (default), "Minecraft Java", "Minecraft Bedrock", "PocketMine-MP", "Nukkit".
- **Placement:** Immediately after the existing status filter in the filter bar row.

### Status Filter (existing — keep)
- **Options:** "All Status" (default), "Running", "Stopped", "Sleeping", "Pending", "Error".
- No changes from current implementation beyond cosmic theme restyling.

### Restart Button (D-07)
- **Visual:** Cyan-accented button using lucide `RotateCcw` icon, inline with View and Start/Stop buttons on each card/row.
- **Accessibility:** Uses `aria-label="Restart {server.name}"` with a tooltip on hover showing "Restart".
- **Interaction:** Click → shows confirmation dialog (cosmic-themed modal overlay) → on confirm, calls `serverStore.restartServer(id)` → shows "Restarting {name}..." toast → on completion, toast updates to "{name} restarted".
- **Disabled state:** Grayed out with cursor-not-allowed when server status is `starting`, `stopping`, or `crashed`.

### 30s Polling (D-09)
- **Behavior:** `setInterval` of 30000ms inside `ServerManagerPage.jsx`. Triggers `useServerStore().fetchServers()`.
- **Cleanup:** `clearInterval` on component unmount.
- **Status change detection:** Compare previous `servers` array statuses with new poll results. For each server whose status changed, call `useUIStore().addToast({ type: 'info', message: '{name} is now {status}' })`.
- **Polling indicator:** No explicit visual indicator. The data refresh is silent. If a poll fails, show the error state defined in Copywriting Contract.

### Table View Columns (from legacy `ServerManager.jsx`)
When in table view, columns in order:
1. **Name** — linked to `/servers/:id`
2. **Game** — capitalized game type
3. **Host:Port** — `public_host || "localhost" : port`
4. **Image** — container image name (truncated with `font-mono text-xs`)
5. **Node** — `node_id.slice(0, 8)...` or "Auto-assigned"
6. **Status** — using `StatusBadge` component
7. **Actions** — View, Restart, Start/Stop/Sleep/Wake

### Card View Content (D-01)
Per card, top to bottom:
1. **Header row:** Server name (left), status dot (right)
2. **Meta row:** Game type | Image name (mono, truncated) | Node ID
3. **Actions row:** View button (link), Restart button, Start/Stop/Sleep/Wake button
