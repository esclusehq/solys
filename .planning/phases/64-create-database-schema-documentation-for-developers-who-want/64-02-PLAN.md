Annotation conventions:
- 1-3 lines of `///` before the struct
- First line: what this entity represents
- Optional second line: where it's stored / how it maps
- Optional third line: cross-reference to important implementation details
- Field-level comments: ONLY add for non-obvious fields (e.g., `status`, `executor_type`, `policy_json`) — skip obvious fields like `id`, `created_at`, `updated_at`, `name`
- Field comments are optional `///` lines between the field declaration and the next field
- Do NOT modify any `#[derive(...)]`, `#[serde(...)]`, or `#[sqlx(...)]` attributes
