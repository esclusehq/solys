    - is_nullable: `c.is_nullable = 'YES'`
    - is_pk: check via `SELECT ku.column_name FROM information_schema.table_constraints tc JOIN information_schema.key_column_usage ku ON tc.constraint_name = ku.constraint_name WHERE tc.constraint_type = 'PRIMARY KEY' AND tc.table_name = $1 AND ku.column_name = c.column_name` — use a subquery approach
    - data_type mapping: implement `fn map_pg_type(pg_type: &str) -> &'static str` that maps:
      - `"uuid"` → `"uuid"`
      - `"text"`, `"character varying"`, `"character"` → `"text"`

      - `"boolean"` → `"bool"`

      - `"timestamp without time zone"` → `"timestamp"`
      - `"jsonb"`, `"json"` → `"jsonb"`
      - `"USER-DEFINED"` → try resolve via `pg_typeof`, fallback to `"enum"`
      - Everything else → lowercase the input

    **`pub async fn get_foreign_keys(pool: &PgPool, table: &str) -> anyhow::Result<Vec<ForeignKey>>`**
    - Use RESEARCH.md lines 177-193 query pattern

    **`pub async fn get_indexes(pool: &PgPool, table: &str) -> anyhow::Result<Vec<IndexInfo>>`**
    - Query: `SELECT indexname, indexdef FROM pg_indexes WHERE schemaname = 'public' AND tablename = $1 ORDER BY indexname`

    **`pub async fn get_enums(pool: &PgPool) -> anyhow::Result<HashMap<String, Vec<String>>>`**
    - Query: `SELECT t.typname, e.enumlabel FROM pg_type t JOIN pg_enum e ON t.oid = e.enumtypid ORDER BY t.typname, e.enumsortorder`
    - Return as `HashMap<typname, Vec<labels>>`

    **`pub async fn get_table_comment(pool: &PgPool, table: &str) -> anyhow::Result<Option<String>>`**
    - Query: `SELECT pgd.description FROM pg_catalog.pg_class pgc LEFT JOIN pg_catalog.pg_description pgd ON pgd.objoid = pgc.oid AND pgd.objsubid = 0 WHERE pgc.relname = $1`

    Use `sqlx::query_as` with typed row structs for each query where the result set is non-trivial. For the column query use a raw `sqlx::query` with `.fetch_all(pool)` and map rows manually since the join produces nullable fields.

          ```
          uuid id PK
          text name
          uuid user_id FK
          ```

          The correct format is: `type name [markers]`:
          `format!("        {} {} {}{}\n", col.data_type, col.name, pk_marker, fk_marker)`
          
          But then `uuid id PK` would render as `uuid id PK` which is correct. And `uuid user_id FK` renders as `uuid user_id FK`. PK and FK together: `uuid id PK FK` (if both). That's fine for Mermaid.

          ```
          uuid id PK
          text name

          let mut markers = String::new();
