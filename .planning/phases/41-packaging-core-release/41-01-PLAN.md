<tasks>

<task type="auto">
  <name>Task 1: Add musl target to Cargo.toml for static builds</name>
  <files>web-agent/Cargo.toml</files>
  <action>
Add the following at the end of Cargo.toml to enable static builds with musl:

```toml
[target.x86_64-unknown-linux-musl]
linker = "musl-gcc"

[profile.release]
opt-level = 3
lto = "fat"
codegen-units = 1
strip = true
overflow-checks = false
```

Add a comment explaining: "# Static build with musl for portable binary"
  </action>
  <verify>
    <automated>grep -q "x86_64-unknown-linux-musl" web-agent/Cargo.toml && grep -q "musl-gcc" web-agent/Cargo.toml</automated>
  </verify>
  <done>musl target configured in Cargo.toml</done>
</task>

<task type="auto">
  <name>Task 2: Build static release binary</name>
  <files>web-agent/target/x86_64-unknown-linux-musl/release/devnode-agent</files>
  <action>
First ensure musl target is installed:
  ```bash
  rustup target add x86_64-unknown-linux-musl
  ```

Then build with musl target:
  ```bash
  cd web-agent && cargo build --release --target x86_64-unknown-linux-musl
  ```

Rename output for clarity:
  ```bash
  cp web-agent/target/x86_64-unknown-linux-musl/release/web-agent web-agent/target/x86_64-unknown-linux-musl/release/devnode-agent
  ```

Verify it's static (no dynamic dependencies):
  ```bash
  file web-agent/target/x86_64-unknown-linux-musl/release/devnode-agent
  ldd web-agent/target/x86_64-unknown-linux-musl/release/devnode-agent 2>&1 | grep -v "not a dynamic"
  ```
  </action>
  <verify>
    <automated>test -f web-agent/target/x86_64-unknown-linux-musl/release/devnode-agent && file web-agent/target/x86_64-unknown-linux-musl/release/devnode-agent | grep -q "statically linked"</automated>
  </verify>
  <done>Static binary built and verified</done>
</task>
