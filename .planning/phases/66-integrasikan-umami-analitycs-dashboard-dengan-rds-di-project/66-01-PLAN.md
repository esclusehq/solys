## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> analytics.esluce.com | Public internet -> Caddy reverse proxy (TLS termination) |
| Caddy -> Umami container | Internal Docker network (umami-net bridge) |
| Umami -> RDS PostgreSQL | EC2 -> RDS via public endpoint with TLS (sslmode=require) |
| Tracking script -> Browser | Served from analytics.esluce.com, loaded by visitor browsers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-66-01 | Information Disclosure | RDS PostgreSQL | mitigate | Security group restricts port 5432 to EC2 SG only (DEPLOYMENT.md Step 1) |
| T-66-02 | Tampering | Umami -> RDS connection | mitigate | sslmode=require in DATABASE_URL; RDS enforces TLS by default |
| T-66-03 | Spoofing | Tracking script delivery | mitigate | Caddy auto-redirects HTTP to HTTPS; Strict-Transport-Security with includeSubDomains |
| T-66-04 | Elevation of Privilege | Umami admin login | mitigate | Strong APP_SECRET via openssl rand -hex 32; password change enforced in Step 5 |
| T-66-05 | Denial of Service | Ad blocker blocking tracking | mitigate | TRACKER_SCRIPT_NAME renamed to analytics.js; COLLECT_API_ENDPOINT renamed to /api/collect |
| T-66-06 | Information Disclosure | Caddy response headers | mitigate | -Server directive removes version; X-Frame-Options DENY prevents clickjacking |
</threat_model>
