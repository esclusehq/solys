## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Incoming webhooks | mitigate | Add HMAC-SHA256 signature verification (D-01) |
| T-01-02 | Repudiation | Config loading | mitigate | Fail fast on missing required values (D-03) |
| T-01-03 | Information Disclosure | JWT secret | mitigate | Reject default/insecure secrets in production (D-03) |
| T-01-04 | Denial of Service | Config validation | accept | Validation failure = startup failure, not runtime crash |

## Security Notes

- Use `subtle::ConstantTimeEq` for signature comparison to prevent timing attacks
- No logging of secrets or signature contents
- Webhook endpoint should return 200 even on verification failure (to avoid retry loops) - but log the failure
