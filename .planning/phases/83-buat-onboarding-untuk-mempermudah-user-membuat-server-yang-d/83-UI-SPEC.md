## Interactions & States

### Visual Focal Point

The primary visual anchor on each wizard step is the step title (Display 24px 600), located at the top of the modal beneath the step progress bar. The user's eye then moves to the step content area (selectable game type cards, resource plan cards, or configuration fields), and finally to the action button at the bottom — either "Next" (Steps 1–3, as a secondary action) or "Deploy Server" (Step 4, as the primary CTA). This Z-pattern top→content→action hierarchy ensures the user always knows where they are, what they can choose, and how to proceed.

### Modal Lifecycle (ServerOnboardingWizard)

| Trigger | Behavior | Source |
|---------|----------|--------|
| Click "Create your first server" on Dashboard empty state | Open ServerOnboardingWizard modal (full-screen overlay, z-50) | D-01, DashboardPage line 237 |
| Click "Skip" | Close wizard, set isOnboarded=true, stay on dashboard | Existing Onboarding.jsx pattern |
| Click backdrop outside modal | Close wizard, no state saved | Standard modal pattern |
| Click "Back" in Step 2/3/4 | Navigate to previous step, preserve selections | Wizard pattern |
| Click "Next" in Step 1/2/3 | Validate current step, advance to next | Wizard pattern |
| Click "Deploy Server" in Step 4 | Call serversApi.create(), show loading state, on success → toast + redirect to /servers/{id} | D-07, D-09 |
| API success | Auto-redirect to server detail page `/servers/{id}` | D-07 |
| API error | Show error toast, remain on Step 4, re-enable button | Standard error pattern |

### Step 1 — Game Type Selection (Type)

| Element | Interaction |
|---------|-------------|
| 3 cards (Java/Bedrock/PocketMine) | Click to select. Selected card gets cyan border + glow. Only one selected at a time. |
| Card icon | lucide-react icons: Java (Cpu chip or custom), Bedrock (Shield/Bricks icon), PocketMine (Smartphone icon) |
| Card structure | Icon (32x32) → Title (Heading 20px) → Description (Body 16px) → "Recommended" badge (optional) |
| Progress dots | Show 4 dots at bottom of step title area, current step highlighted cyan |
| Default selection | Java Edition pre-selected (most common first choice) |

### Step 2 — Resource Plan Selection (Resources)

| Element | Interaction |
|---------|-------------|
| Plan cards (Free/Hobby/Pro) | Click to select. Show feature list per plan. Selected card gets cyan border. |
| Resource sliders | Appear after plan selection. RAM/CPU/Disk constrained to plan limits. Slider or stepper controls. |
| Plan feature display | Feature list with check icons |
| Plan price display | Monthly price displayed if applicable (Free shows "Free") |

### Step 3 — Server Configuration (Config)

| Element | Interaction |
|---------|-------------|
| Server Name input | Text input, required, max 64 chars |
| Minecraft Version dropdown | Auto-populated from MINECRAFT_VERSIONS (ref from CreateServerModal), filtered by selected game type |
| Port input | Text input, numeric only. Auto-suggested random port on load. Placeholder shows "Random" |
| Template dropdown | Optional. Populated from existing template API (reference CreateServerModal.jsx) |

### Step 4 — Review & Deploy

| Element | Interaction |
|---------|-------------|
| Review summary | Read-only display of all selections from Steps 1-3 |
| "Deploy Server" button | Primary CTA. On click: show disabled+loading state ("Deploying…"), call serversApi.create() |
| Loading state | Button shows spinner + "Deploying…". Disable all interactions |
| Success | Toast: "Server created successfully!" → 1.5s delay → navigate to `/servers/{id}` |
| Failure | Toast: error message. Re-enable button. User can fix and retry |

### States Summary

| State | Behavior |
|-------|----------|
| Empty (Dashboard) | Existing empty state with "Create your first server" button — unchanged |
| Wizard open | Full modal overlay, backdrop, 4-step progress bar |
| Step active | Current step content visible, progress bar shows position |
| Step loading | (Not applicable — data loads from existing stores/hooks) |
| Deploying | Button disabled + spinner, all inputs read-only |
| Success | Toast + auto-redirect to server detail |
| Error | Toast with error detail, button re-enabled, inputs editable |
| Wizard closed (skip) | Modal dismissed, no server created, dashboard remains |
