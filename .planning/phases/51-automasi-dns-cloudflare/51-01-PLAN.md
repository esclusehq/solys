## Goal
Backend foundation untuk Cloudflare DNS automasi: database storage, settings repository, API endpoints, dan WebSocket protocol extension.

## Tasks

### 1. Database — Seed `cloudflare_config` di `app_settings`
- Insert seed row `INSERT INTO app_settings (key, value, updated_at) VALUES ('cloudflare_config', '{}'::jsonb, NOW()) ON CONFLICT (key) DO NOTHING;`
- Kolom `value` JSONB berisi: `{ "api_token": "", "zone_id": "", "zone_name": "esluce.com", "wildcard_domain": "esluce.com", "auto_refresh": true, "refresh_interval_secs": 300 }`

### 2. Domain Entity — `CloudflareConfig`
- File: `api/src/domain/entities/cloudflare_settings.rs`
- Struct `CloudflareConfig` dengan fields: `api_token` (String, masked di GET), `zone_id` (String), `zone_name` (String), `wildcard_domain` (String), `auto_refresh` (bool), `refresh_interval_secs` (u64)
- Method `is_configured()` → bool (cek api_token + zone_id tidak kosong)
- Method `masked_api_token()` → tampilkan 4 karakter pertama + "..."

### 3. Settings Repository — Extend Trait
- File: `api/src/domain/repositories/settings_repository.rs`
- Tambah: `async fn get_cloudflare_config(&self) -> Result<CloudflareConfig>`
- Tambah: `async fn save_cloudflare_config(&self, config: &CloudflareConfig) -> Result<()>`

### 4. Postgres Settings Repository — Implement
- File: `api/src/infrastructure/repositories/postgres_settings_repository.rs`
- Implement `get_cloudflare_config`: query `app_settings WHERE key = 'cloudflare_config'`, deserialize JSONB ke `CloudflareConfig`, return default jika tidak ada
- Implement `save_cloudflare_config`: `INSERT ... ON CONFLICT (key) DO UPDATE`

### 5. Settings Handlers — Cloudflare Endpoints
- File: `api/src/presentation/handlers/settings_handlers.rs`
- Tambah `get_cloudflare_config` — GET, mask api_token di response
- Tambah `save_cloudflare_config` — PUT, jika api_token kosong di payload, keep existing
- Response format: `{ "success": true, "data": { ... } }`

### 6. Route Registration
- File: `api/src/presentation/routes/api_routes.rs`
- Tambah: `.route("/api/v1/settings/cloudflare", get(get_cloudflare_config).put(save_cloudflare_config))`

### 7. Container — Register Cloudflare Settings Handler
- File: `api/src/bootstrap/container.rs`
- Settings repository sudah registered sebagai `settings_repository` — cukup pass ke handler function yang baru

### 8. WebSocket Protocol — Extend `BackendToAgent`
- File: `agent/agent-core/crates/agent-proto/src/messages.rs`
- Tambah variant di `BackendToAgent`: `DnsConfig(DnsConfigPayload)` — backend kirim Cloudflare config ke agent
- Tambah struct `DnsConfigPayload` dengan fields: `api_token`, `zone_id`, `zone_name`, `wildcard_domain`, `auto_refresh`, `refresh_interval_secs`, `public_ip`
- Tambah variant di `AgentToBackend`: `DnsStatus(DnsStatusPayload)` — agent report DNS record status
- Tambah struct `DnsStatusPayload` dengan fields: `domain`, `record_type`, `record_id`, `ip`, `status` (created/updated/deleted/error), `message`

## Verification
- `GET /api/v1/settings/cloudflare` returns config with masked token
- `PUT /api/v1/settings/cloudflare` saves config, empty token = keep existing
- WebSocket `DnsConfig` message can be sent from backend to agent
- WebSocket `DnsStatus` message can be sent from agent to backend
- Tests pass: `cargo test -p api`
