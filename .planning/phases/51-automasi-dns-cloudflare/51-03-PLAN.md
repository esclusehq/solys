## Goal
Dashboard settings page untuk Cloudflare DNS: input API token + zone ID, view/manage DNS records, lihat status domain per node.

## Tasks

### 1. API Client — `cloudflareApi`
- File: `app/src/lib/api.js`
- Tambah objek `cloudflareApi`:
  - `getConfig()` → `GET /api/v1/settings/cloudflare`
  - `saveConfig(data)` → `PUT /api/v1/settings/cloudflare`

### 2. Cloudflare Settings Tab
- File baru: `app/src/components/settings/CloudflareSettings.jsx`
- Form fields:
  - `Cloudflare API Token` (password field, masked)
  - `Zone ID` (text field)
  - `Zone Name` (display only, dari GET response)
  - `Wildcard Domain` (display only)
  - `Auto Refresh IP` (toggle switch)
  - `Refresh Interval` (number input, seconds, muncul jika auto-refresh on)
- Tombol "Save" dan "Test Connection"
- Test Connection: try `GET /zones/{zone_id}` via backend proxy endpoint (or langsung dari backend)
- Status indicator: "Connected" / "Invalid Token" / "Not Configured"
- Loading state, error handling, toast messages

### 3. Wrap API Token
- Di backend handler, api_token di-mask di GET response (show first 4 chars + "****")
- Di form, tampilkan "••••••••••" jika sudah ada token tersimpan

### 4. Register Tab di SettingsPage
- File: `app/src/pages/settings/SettingsPage.jsx`
- Import `CloudflareSettings` component
- Tambah tab `{ id: 'cloudflare', label: 'Cloudflare DNS' }` di array `tabs`
- Render `{activeTab === 'cloudflare' && <CloudflareSettings />}`

### 5. DNS Status Display (per Node)
- File: `app/src/components/settings/CloudflareSettings.jsx`
- Section "DNS Records" — list semua node yang punya DNS record
- Untuk setiap node: domain name, IP yang terdaftar, status (active/pending/error), last updated
- Data dari backend: `GET /api/v1/nodes/:id/dns` (atau via status WebSocket)

### 6. Custom Subdomain Input (Future-Ready)
- Saat node registration atau edit, input field untuk custom subdomain
- Validasi: hanya huruf kecil, angka, dan dash
- Preview: `{subdomain}.esluce.com`

## Verification
- Cloudflare DNS tab visible di Settings page
- API token dapat di-save dan di-load (masked)
- Test Connection button works (menunjukkan koneksi valid/invalid)
- Auto-refresh toggle works (simpan/load konfigurasi)
- DNS records section menampilkan status per node (data dari backend)
- Build: `cd app && npm run build` sukses tanpa error
