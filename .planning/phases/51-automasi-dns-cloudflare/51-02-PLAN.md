## Goal
Agent dapat berkomunikasi dengan Cloudflare API untuk manage DNS records: create wildcard record, manage subdomain per-node, dan auto-refresh IP (DDNS-like).

## Tasks

### 1. Agent Config — Tambah Opsi DNS
- File: `agent/agent-core/crates/agent-config/src/lib.rs` (or config struct definitions)
- Tambah field `dns_enabled: bool` (default: true) di config struct
- Tambah field `dns_refresh_interval_secs: u64` (default: 300)

### 2. DNS Handler — `handlers/dns.rs`
- File baru: `agent/solys/src/handlers/dns.rs`
- Task types:
  - `"dns.configure"` — terima `CloudflareConfig` dari backend via payload
  - `"dns.create_record"` — buat DNS record A untuk IP agent
  - `"dns.update_record"` — update DNS record jika IP berubah
  - `"dns.delete_record"` — hapus DNS record
  - `"dns.status"` — report status DNS records ke backend

### 3. Cloudflare API Client
- Module internal di `handlers/dns.rs` atau file `handlers/cloudflare_client.rs`
- Functions:
  - `create_dns_record(api_token, zone_id, name, ip)` → POST `/client/v4/zones/{zone_id}/dns_records` — buat record A
  - `update_dns_record(api_token, zone_id, record_id, name, ip)` → PATCH `/client/v4/zones/{zone_id}/dns_records/{record_id}` — update IP
  - `delete_dns_record(api_token, zone_id, record_id)` → DELETE `/client/v4/zones/{zone_id}/dns_records/{record_id}`
  - `list_dns_records(api_token, zone_id, name)` → GET `/client/v4/zones/{zone_id}/dns_records?name={name}` — cari existing record
- Error handling: retry untuk rate limit (HTTP 429), timeout
- Gunakan `reqwest` (already in deps) untuk HTTP calls

### 4. IP Detection — `handlers/dns_watch.rs`
- File baru: `agent/solys/src/handlers/dns_watch.rs`
- `DnsWatcher` struct:
  - State: `current_ip: Arc<RwLock<String>>`, `cloudflare_config: Arc<RwLock<Option<CloudflareConfig>>>`
  - `start()` — spawn background task periodic check
  - `check_ip()` — deteksi public IP via external API (e.g., `https://api.ipify.org`, `https://checkip.amazonaws.com`, fallback chain)
  - `on_ip_change(old_ip, new_ip)` — trigger DNS record update via Cloudflare API
- Interval: default 300 detik, configurable

### 5. Register Handler di `handlers/mod.rs`
- File: `agent/solys/src/handlers/mod.rs`
- Tambah `pub mod dns;`
- Tambah `pub mod dns_watch;`
- Tambah task type dispatch:
  - `"dns.configure"` → `dns::handle_configure`
  - `"dns.create_record"` → `dns::handle_create`
  - `"dns.update_record"` → `dns::handle_update`
  - `"dns.delete_record"` → `dns::handle_delete`
  - `"dns.status"` → `dns::handle_status`
- Task config untuk DNS operations (timeout 30s, max_retries 3 untuk rate limit)

### 6. Integrate DNS Watcher di Agent Startup
- File: `agent/solys/src/main.rs`
- In `run_agent_core()`, spawn `DnsWatcher` setelah runtime detection
- `dns_watcher.start()` sebagai background task

### 7. Agent Connection — Handle `DnsConfig` Message
- File: `agent/solys/src/agent_connection.rs`
- Handle `BackendMessage::DnsConfig` — update config di DnsWatcher
- Kirim task `dns.configure` ke handler

### 8. DNS Record Naming Convention
- Default: `{node_name}.esluce.com` → record A ke IP agent
- Custom: user bisa pilih nama subdomain dari dashboard (di-apply via `dns.configure` payload)
- Record TTL: 120 (auto)

## Verification
- Agent receives `DnsConfig` from backend and stores it
- Agent detects public IP correctly via multiple fallback APIs
- Agent creates DNS A record at `{node_name}.esluce.com` pointing to agent IP
- Agent detects IP change and updates DNS record automatically
- Agent reports DNS status back to backend via `DnsStatus` message
- Cloudflare API error handling works (rate limit, auth failure)
- Build: `cargo build -p solys`
