## Interaction Contract

### View Toggle (D-02, D-03)
- **Visual:** Segmented button group with two icon buttons: `LayoutGrid` icon (card view) and `List` icon (table view). Active state uses `var(--color-cosmic-cyan)` text/border. Inactive uses `var(--color-text-muted)`.
- **Accessibility:** Each toggle button uses `aria-label` — `aria-label="Card view"` and `aria-label="Table view"`.
- **Persistence:** View preference saved to `localStorage` key `nodeViewMode` with values `'card'` or `'table'`.
- **Default:** `'card'` (current default view matching Phase 75).
- **Placement:** Right side of the node list panel header, next to the node count badge.
- **Focal point:** The node list is the primary visual anchor. The + Add Node button is the primary action CTA in the page header top-right.

### Split-Panel Layout (D-01)
- **Left panel** (flex-[4]): Node list with view toggle, node count badge, node cards/rows
- **Right panel** (flex-[6]): Node detail panel with 3 tabs (Overview, API Keys, Tokens)
- **Responsive:** On viewports <1024px, the detail panel stacks below the node list (full-width each)
- Gap between panels: `gap-6` (24px)

### Node Card View Content (card mode, D-04)
Per card, top to bottom:
1. **Header row:** Node name (left), status emoji + text (right)
2. **Meta row 1:** IP address (mono) | Memory (GB) | CPU cores
3. **Meta row 2:** Uptime (relative time, e.g. "2h 30m") | Last seen (relative time, e.g. "5m ago")
4. **Status:** Click selects node → opens details in right panel

### Node Table View Columns (agent's discretion — mirror card fields)
When in table view, columns in order:
1. **Name** — clickable (selects node for detail panel)
2. **IP Address** — `font-mono text-xs`
3. **Memory** — GB formatted (`(node.total_memory / 1048576).toFixed(1)`)
4. **CPU** — core count + "C" suffix
5. **Uptime** — relative time format
6. **Last Seen** — relative time format (e.g. "2m ago", "1h ago")
7. **Status** — emoji + text (same pattern as card view)

### Detail Panel — Overview Tab (D-09)
**Health Status section** (4 metrics, visual refresh per agent's discretion):
- **Status** — color-coded label (green=healthy, orange=warning, red=unhealthy) with colored dot indicator
- **CPU** — progress bar showing `health.cpu_usage` percentage. Color coding: green (<70%), orange (70-90%), red (>90%)
- **Memory** — progress bar showing used vs total. Show as "X MB / Y MB" label above bar. Color coding: green (<70%), orange (70-90%), red (>90%)
- **Containers** — count display with icon. Can use a small mini-bar showing active/total.
- **Layout:** 4-column grid (same as existing `grid-cols-4 gap-4` pattern from Nodes.jsx line 451). Existing `bg-[rgba(0,0,0,0.2)] p-3 rounded-lg` card styling.
- **Loading:** Skeleton pulse animation (4 items in grid).

**System Info section** (keep existing):
- Agent Version, OS, Memory, CPU Cores, Container Runtime
- 2-column grid layout with `bg-[rgba(0,0,0,0.2)] p-3 rounded-lg` cards

### Detail Panel — API Keys Tab (keep existing)
- Key list with name, created date, active/inactive badge, delete button
- "Generate" button + optional key name input
- No functional changes — visual alignment with cosmic theme only if not already matching

### Detail Panel — Tokens Tab (keep existing)
- "Get Installation Token" button (cosmic-purple accent)
- Token expiry selector modal (1h/24h/3d/7d)
- Token display modal with installation guide
- No functional changes — visual alignment with cosmic theme only

### Uptime/Last Seen Formatting (D-04)
- **Uptime:** Format as relative duration (e.g., "2h 30m", "1d 4h", "<1m" if <1 minute). Derive from node data fields if available (e.g., `connected_at` or `last_heartbeat`).
- **Last seen:** Format as relative time from now (e.g., "just now", "5m ago", "1h ago", "2d ago"). Use `node.last_seen_at` or calculate from `node.updated_at`.
- **Placement in card view:** Below IP/Memory/CPU row, as two inline text items.
- **Placement in table view:** Dedicated columns #5 and #6.

### Node Limit Badge (keep existing from Nodes.jsx line 146)
- In node list panel header, right side
- Shows "X / Y Nodes" where Y is plan limit (or ∞ for unlimited)
- Color coding: green (< 1 away from limit), yellow (1 away from limit), red (at limit)
- Does not affect the + Add Node button behavior (existing logic preserved)

### Modals (keep existing)
- **Add Node Modal:** Name input, "An API key will be generated after creation" helper text, Cancel/Create buttons. Cosmic-cyan border accent.
- **Node Created (API Key) Modal:** Shows generated key (orange highlight), copy button, installation instructions (Linux, Docker, Windows, Config file). Done button.
- **API Key Generated Modal:** Shows generated key (orange highlight), copy button. Done button.
- **Token Expiry Modal:** Select expiry dropdown (1h/24h/3d/7d), Cancel/Generate buttons. Cosmic-purple border accent.
- **Token Display Modal:** Shows generated token (purple highlight), copy button, installation guide steps. Cosmic-purple border accent.

### Loading States
- **Initial load (first fetch):** `EscluseSpinner` centered in the node list panel (size=80, color="#06b6d4") — matching existing pattern from Nodes.jsx line 158
- **Subsequent loads (refetch):** `SkeletonText` with 5 lines — matching existing pattern from Nodes.jsx line 162
- **Detail panel health loading:** Skeleton pulse animation — 4 skeleton cards in grid layout — matching existing pattern from Nodes.jsx line 443
- **Detail panel keys loading:** "Loading..." text — matching existing pattern from Nodes.jsx line 543

### Empty State
- When `nodes.length === 0` AND not loading:
  - Centered text: "No nodes found"
  - "Add Local Node" button below the text (creates node on 127.0.0.1 for dev workflow)
  - Query parameter `?local=true` could be considered for future enhancement

### Health Metrics 30s Polling (existing from useNodeHealth.js)
- `setInterval` of 30000ms inside `useNodeHealth()` hook. Triggers `refetch()`.
- `clearInterval` on node ID change or component unmount.
- **Polling indicator:** No explicit visual indicator. Data refresh is silent.
- If a poll fails, keep last known health data (don't clear it). Show stale data with a subtle warning.
