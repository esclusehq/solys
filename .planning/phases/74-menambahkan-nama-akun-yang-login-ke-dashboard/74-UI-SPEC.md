## Components

### 1. TopBar (new component — `app/src/components/TopBar.jsx`)

**Placement:** Inside `Layout.jsx`, between the opening `<main>` tag and the `<Outlet />`.

```
┌──────────────────────────────────────────────────────────┐
│  Sidebar                      │  [Avatar] [Display Name] ▼ │  ← TopBar (56px)
│                               ├──────────────────────────┤
│                               │                          │
│                               │     <Outlet />           │
│                               │     (page content)       │
│                               │                          │
└───────────────────────────────┴──────────────────────────┘
```

**Structure (left to right):**
- Empty flex spacer (pushes user info to the right edge)
- User area: circular avatar (32px) + display_name text (14px font-medium)
- Click on the user area OR the chevron icon opens a dropdown menu

**States:**

| State | Visual | Behavior |
|-------|--------|----------|
| Default | Shows avatar + display_name. If no avatar_url: initial-letter fallback in a `--color-cosmic-purple` circle. If no display_name: show email prefix. | Click opens dropdown |
| Hover (avatar) | Subtle brighten/border glow using `--color-cosmic-cyan` | - |
| Dropdown open | Dropdown menu visible below the user area | Click outside or press Escape closes dropdown |

**Dropdown menu items:**
1. **Profile** — lucide-react `User` icon → navigates to `/settings`
2. **Settings** — lucide-react `Settings` icon → navigates to `/settings`
3. **Divider** — thin border (border-[var(--color-cosmic-border)])
4. **Logout** — lucide-react `LogOut` icon, text colored `--color-cosmic-red` → calls `useAuthStore.logout()`

Layout.jsx modification: Add `<TopBar />` above `<Outlet />` inside `<main>`.

### 2. Welcome Message (modify existing `app/src/pages/dashboard/DashboardPage.jsx`)

**Current behavior:** `getWelcomeMessage()` uses `user?.name` (wrong field).

**Change:** Use `user.display_name` with fallback to `user.email?.split('@')[0]`.

```
│  Welcome back, John!                                  │  ← 24px font-medium, white

├──────────────────────────────────────────────────────┤

```

| State | Copy | Notes |

| display_name empty | Fallback to email prefix | Matches sidebar behavior |

### 3. Sidebar User Area (no change — keep as-is per CONTEXT.md deferred)

The existing sidebar user area at lines 100-121 of `Sidebar.jsx` remains unchanged per CONTEXT.md deferring the click behavior unification decision.

## Interaction Contract

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Open dropdown | Click on user area (avatar + name) OR chevron icon in TopBar | Dropdown menu fades in below the trigger; slight downward offset (4px gap) |
| Close dropdown | Click outside dropdown | Dropdown fades out immediately |
| Close dropdown | Press Escape | Dropdown fades out immediately |
| Close dropdown | Click on a menu item | Dropdown closes after navigation/logout |
| Navigate (Profile/Settings) | Click menu item | `navigate('/settings')` — dropdown closes |
| Logout | Click "Logout" menu item | Calls `useAuthStore.logout()` → redirect follows existing auth flow (no confirmation dialog needed — matches existing logout pattern) |
| Avatar hover | Mouse over avatar | Subtle cyan border glow |
| Display name truncation | Name exceeds available width | `truncate` class with max-width constraint |

**Animation:** Dropdown uses Tailwind transitions (opacity + translateY, 150ms ease-out when opening, 100ms ease-in when closing).

**Logout confirmation:** NO confirmation dialog per CONTEXT.md deferred decision. This matches the existing logout pattern in the app (no confirmation on sidebar or settings logout).

## Loading / Empty / Error States

| State | Element | Behavior |
|-------|---------|----------|
| Loading | TopBar user area | No special loader needed — authStore.user resolves before Dashboard loads. If user is somehow null, TopBar renders nothing (conditional render on `user`) |
| Loading | Welcome message | Existing `EscluseSpinner` on DashboardPage handles initial load |
| Error | TopBar user area | No error state — if auth fails, user is redirected to login by `ProtectedRoute`. TopBar only renders when user exists. |
| Null user | TopBar | Render nothing — `<div>` is conditionally displayed only when `user` is truthy |
| No avatar_url | Avatar circle | Initial-letter fallback: first character of `display_name` or email prefix, uppercase, in a `--color-cosmic-purple` tinted circle |
| No display_name | Display text | Fallback to `user.email?.split('@')[0]` or 'User' |
