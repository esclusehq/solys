## Focal Point

The primary visual anchor on the Backup tab is the **config panel** (positioned ABOVE the existing backup history table in `ServerBackups.jsx`). Visual hierarchy flows top-to-bottom:

1. **Auto-backup toggle** — the first and most prominent control. Determines whether the rest of the config is visible/enabled.
2. **Schedule selector** — preset dropdown with custom cron expression option.
3. **Retention rules** — max count input + label-based retention preset dropdown.
4. **Storage provider** — provider selector (local/S3) + S3 profile dropdown (shown conditionally).
5. **Save Changes button** — the single primary action for the config panel.
6. **Backup History table** — existing read-only table below the config panel, separated by `mt-8` gap.

This order follows the user's mental model: enable → configure schedule → configure retention → configure storage → save → view history.

## Component Inventory

### 1. ServerBackupConfig (NEW)

- **File:** `app/src/components/ServerBackupConfig.jsx`
- **Type:** Inline form panel rendered ABOVE the table in `ServerBackups.jsx`
- **Purpose:** Configure per-server automated backup schedule, retention, and storage provider.
- **Position:** Inside the Backup tab container `max-w-5xl`, first child before the "Backup History" header section.

#### Form Fields (in order)

| Field | Type | Options / Validation | Visual |
|-------|------|---------------------|--------|
| Enable Auto Backup | Toggle switch | On/off | `w-12 h-6 rounded-full` toggle, cyan when active. Disables all other controls when off. |
| Schedule Preset | Dropdown select | "Every 6 hours", "Every 12 hours", "Daily (midnight)", "Weekly (Sunday)", "Monthly (1st)", "Custom..." | Full-width select with `bg-[var(--color-cosmic-card)]/60` styling |
| Custom Cron Expression | Text input (font-mono) | Visible only when "Custom..." selected from preset. Validates cron syntax. | `w-full font-mono` input with placeholder `0 */6 * * *` |
| Max Backups Retained | Number input | Min: 1, Max: 100, Default: 10 | `w-full` number input |
| Label Retention Preset | Dropdown select | "None (count only)", "Keep 7 daily, 4 weekly, 3 monthly", "Keep 14 daily, 6 weekly, 6 monthly", "Keep 30 daily, 8 weekly, 12 monthly" | Full-width select |
| Storage Provider | Dropdown select | "Local Storage" (value: `local`), "S3-Compatible" (value: `s3`) | Full-width select with emoji prefix |
| S3 Profile | Dropdown select | Populated from `GET /settings/s3/profiles`. Visible only when provider="s3". Shows "No S3 profiles configured" if list is empty. | Full-width select |
| Save Changes | Button | Disabled when unchanged or saving | `w-full` cyan-styled button, shows "Saving..." during submission |

- **Interaction:**
  - Toggle off → collapse/hide all other controls (keep labels visible but disabled/grayed out)
  - Toggle on → enable all controls
  - "Custom..." preset → reveal custom cron input, hide preset description
  - Provider = "s3" → show S3 profile dropdown. Provider = "local" → hide S3 profile dropdown
  - Save → `PUT /servers/:id/backup-config` with all fields
  - On save success: toast "Backup configuration saved" + refetch config
  - On save error: toast error message

- **States:**

| State | Visual | Behavior |
|-------|--------|----------|
| **Loading** | 4-5 skeleton rows matching form field heights | `skeleton-pulse` animation, `rounded-lg` blocks |
| **Configured / Enabled** | All fields populated, toggle ON, save button enabled | Shows current config values |
| **Configured / Disabled** | Toggle OFF, all other fields grayed out (`opacity-40 pointer-events-none`) | Shows current config values but read-only |
| **Not configured (first time)** | Toggle OFF, defaults shown: max=10, retention preset="Keep 7 daily...", provider="Local Storage" | All fields show defaults |
| **Custom cron active** | Preset shows "Custom...", cron text input visible below preset | Typing in cron input updates the schedule |
| **Saving** | Save button shows "Saving..." spinner, all inputs disabled | Prevents double-submit |
| **Save Error** | Toast: red error message "Could not save backup configuration. {reason}" | Config panel remains open with current values |
| **Save Success** | Toast: green success message "Backup configuration saved" | Config panel reflects saved values |
| **Provider = S3 (no profiles)** | S3 profile dropdown shows "No S3 profiles configured" (disabled) | Settings link opens SystemSettings S3 tab |
| **Free plan** | Yellow upgrade notice banner above toggle (matching existing Settings tab pattern) | "Scheduled backups require a Hobby or Pro plan. Upgrade to enable automatic backup scheduling." + "Upgrade Now →" button |

The existing `useBackups.js` hook handles the backup history table. The new config hook is separate. No modifications needed to the existing hook.

**Destructive actions in this phase:**
1. **Delete backup from history** — existing, uses `confirm('Delete this backup permanently?')`. No changes. Not in new config panel.
2. **Disable auto-backup** — toggle OFF does NOT prompt confirmation. The toggle is immediate. If auto-backup was active, the backend disables the cron task. Not destructive (data is not deleted, only scheduling stops).

**Empty state for the config panel:** Not applicable — config panel always shows either defaults or loaded values. It never has an empty state (the toggle being off IS the "no config" state and is visually handled via disabled fields).

**Error state for the config panel:** Form-level error via toast. Field-level validation uses native HTML5 validation (min/max for number input, pattern for cron expression). No inline field errors.
