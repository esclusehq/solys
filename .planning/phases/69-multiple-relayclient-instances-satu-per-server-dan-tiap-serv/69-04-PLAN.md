<objective>
Add three per-server enhancements to `relay_client.rs`: (1) heartbeat staggering with 0-10s random jitter to prevent thundering herd when many per-server loops restart simultaneously, (2) per-server bandwidth accounting via the `bytes_transferred: Arc<AtomicU64>` already added to `PerServerRuntime` in 69-02, and (3) per-server state transition logging so each server's lifecycle is visible in agent logs.

Purpose:
- **Heartbeat staggering:** Without jitter, N servers starting simultaneously would send N control-stream heartbeats within the same millisecond. With 0-10s random stagger, the effective rate is spread over 10 seconds (e.g., ~6 servers/sec at 60-slot collision rate on a 16M namespace).
- **Bandwidth accounting:** Already set up in 69-02 — this plan adds any remaining wiring (exposing the counter for log/metrics) and documents where the counter is incremented.
- **State transition logging:** Connecting/disconnected/reconnecting/rekeying transitions are currently silent in per-server mode if the outer loop runs without log context for each server. We add structured `tracing::info!` / `tracing::warn!` at each transition with `server_id` span context.

Output:
- `relay_client.rs`: heartbeat jitter, wired bytes_transferred, state transition log lines
</objective>
